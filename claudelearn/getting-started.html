起動と基本

参照: Overview / CLI Docs

Claude Codeを使い始めるために必要なインストール方法、起動コマンド、認証の仕組みを解説します。

インストール方法

環境に合わせて、以下のいずれかの方法でインストールできます。

ネイティブインストール（推奨）

公式インストーラーを使用する最も簡単な方法です。自動アップデートに対応しています。

curl -fsSL https://claude.ai/install.sh | sh
Homebrew（macOS）

macOSのパッケージマネージャーHomebrewを使用する方法です。

brew install claude-code
WinGet（Windows）

Windows用のパッケージマネージャーWinGetで導入できます。

winget install Anthropic.ClaudeCode
npm（非推奨）

npmでのインストールは非推奨です。ネイティブインストーラーはより高速で、依存関係が不要で、自動更新に対応しています。

npm install -g @anthropic-ai/claude-code

起動方法

Claude Codeには3つの主要な起動パターンがあります。用途に応じて使い分けましょう。

1
インタラクティブモード
対話的にClaude Codeを使用する標準的な起動方法。ターミナルでセッションが開始されます。
claude
2
初期プロンプト付き起動
起動と同時にプロンプトを送信します。セッションは維持されるため、続けて対話できます。
claude "このプロジェクトの構成を教えて"
3
パイプモード（非対話）
-p（print）フラグで起動。対話セッションを開かず、結果をstdoutに出力して即座に終了します。シェルスクリプトの中でClaudeを呼んだり、他のコマンドとパイプで繋げるときに使います。
claude -p "package.jsonの依存関係を一覧にして"
cat error.log | claude -p "このエラーの原因を要約して"
claude -p "mainとの差分をレビューして" --output-format json > review.json

セッションの継続・再開

前回のセッションを引き継いで作業を続けることができます。

claude -c（continue）

直前のセッションを自動的に継続します。作業を中断して再開したい場合に便利です。

claude -r（resume）

過去のセッション一覧からピッカーで選択して再開します。複数のセッションを管理している場合に使います。

アップデートと認証

claude update

Claude Codeを最新バージョンに更新します。新機能やバグ修正が頻繁にリリースされるため、定期的な更新を推奨します。

claude auth login

Anthropicアカウントまたはサードパーティプロバイダーでログインします。初回起動時に自動で案内されます。

claude auth logout / claude auth status

ログアウト、または現在の認証状態を確認します。複数アカウントの切り替え時に利用します。

インタラクティブモードの操作

参照: Interactive Mode

インタラクティブモードではキーボードショートカットや特殊入力で効率的に操作できます。全ての操作を覚える必要はありませんが、主要なものを知っておくと生産性が大幅に向上します。

一般的なコントロール

セッション全体の制御に使うショートカットです。

Ctrl+C

現在のレスポンス生成を中断します。長い応答を止めたい場合や、方向性を変えたい場合に使います。

Ctrl+D

セッションを終了してターミナルに戻ります。/exit と同じ動作です。

Ctrl+R

プロンプト履歴を検索します。過去に入力したコマンドを再利用できます。

Ctrl+V

クリップボードから画像を貼り付けます。スクリーンショットをそのままClaude Codeに渡せます。

テキスト編集

Claude Codeの入力欄では、普段のターミナルと同じreadlineキーバインドがそのまま使えます。特別に覚える必要はありません。

Ctrl+A

カーソルを行頭に移動します。長いプロンプトの先頭に戻りたいときに。

Ctrl+E

カーソルを行末に移動します。末尾に追記したいときに。

Ctrl+W

カーソルの手前の1単語を削除します。書き直しが速くなります。

Ctrl+U

カーソルから行頭までを一括削除します。入力をやり直したいときに。

Ctrl+K

カーソルから行末までを一括削除します。後半だけ書き直したいときに。

Ctrl+H

カーソルの手前の1文字を削除します。Backspaceと同じ動作です。

Ctrl+F

カーソルを1文字右に移動します。右矢印キーと同じです。

Ctrl+B

カーソルを1文字左に移動します。左矢印キーと同じです。

Ctrl+P / Ctrl+N

入力履歴を前後に移動します。上下矢印キーと同じです。過去のプロンプトを再利用できます。

Ctrl+T

カーソル前後の2文字を入れ替えます。タイプミスの修正に便利です。

複数行入力

プロンプトに改行を含めたい場合のキー操作です。環境によって使えるキーが異なります。

\ + Enter

バックスラッシュの後にEnterで改行を挿入します。最も確実な方法です。

Option+Enter（macOS）

macOS環境での改行入力です。

Shift+Enter

一部のターミナルで改行として機能します。

クイックコマンド

特殊な接頭辞を使って素早く操作できます。

/
スラッシュコマンド
/ で始めるとコマンド一覧が表示されます。設定変更、セッション管理、各種操作に対応します。
!
Bashモード
! で始めるとシェルコマンドを直接実行します。例：! git status でgitの状態を確認。Claudeを介さず即座に実行されます。
@
ファイルメンション
@ でファイルやディレクトリを参照します。例：@src/index.ts を修正して。タブ補完も使えます。

その他の重要なショートカット

Shift+Tab

パーミッションモードを順番に切り替えます（default → acceptEdits → plan → dontAsk）。現在のモードがステータスバーに表示されます。

セッション管理

参照: Interactive Mode / Workflows

Claude Codeのセッション管理機能を使いこなすことで、作業の継続性を保ち、効率的に開発を進められます。

セッションの命名

セッションに名前を付けることで、後から簡単に見つけて再開できます。

起動時に命名
claude -n "認証機能の実装"

起動と同時にセッション名を設定します。

セッション中に命名
$ /rename バグ修正-issue-42

セッション中に名前を付けたり変更したりします。

セッションの再開方法

1
claude --continue（claude -c）
直前のセッションを自動的に継続します。最も手軽な方法です。
2
claude --resume（claude -r）
セッションピッカーが表示され、過去のセッション一覧から選択できます。
3
claude --from-pr 123
特定のPR番号に関連するセッションを再開します。PR駆動の開発フローで便利です。

セッションピッカーのショートカット

claude -r で表示されるセッションピッカーでは、以下のキーで操作できます。

Up/Down

セッション選択の移動

Enter

選択したセッションを再開

P

セッションのプレビュー

R

セッション名の変更

/

セッション名を検索

A

全プロジェクトのセッション表示

B

ブランチでフィルタ

会話の管理コマンド

/clear vs /compact vs /rewind

/clear

会話履歴を完全に消去して新規セッションを開始。コンテキストをリセットしたい場合に使用します。

/compact

会話をAIが要約して短縮。コンテキストウィンドウを節約しつつ、文脈を維持したい場合に使用します。

/rewind

直前のやりとりを取り消します。意図しない変更を元に戻したい場合に使用します。

/btw でサイドクエスチョン

/btw コマンドを使うと、現在の作業コンテキストを保ったまま「ちょっとした質問」を挟めます。 メインのタスクの流れを中断せずに、関連する疑問を解決できます。例えば、実装中に「ところでこの関数のベストプラクティスは？」といった質問をする場合に最適です。

ファイル参照と入力

参照: Common Workflows

Claude Codeにコンテキストを効率的に渡す方法を解説します。適切なコンテキストを与えることが、質の高い出力を得る鍵です。

@メンション（ファイル・ディレクトリ参照）

@ を入力するとファイルやディレクトリを参照できます。タブ補完にも対応しています。

@src/index.ts のエラーハンドリングを改善して
特定のファイルを指定してタスクを依頼
@src/components/ 配下のコンポーネントを分析して
ディレクトリを指定して複数ファイルを対象にする

MCPリソースの参照

@ はMCPサーバーが提供するリソースも参照できます。例えば、MCPで接続したデータベースやAPIのスキーマ情報などを直接参照可能です。

画像入力

Claude Codeはマルチモーダル対応で、画像を直接入力できます。

ドラッグ&ドロップ

ターミナルに画像ファイルをドラッグ&ドロップします。スクリーンショットやUIモックアップの分析に便利です。

Ctrl+V でペースト

クリップボードにコピーした画像を貼り付けます。スクリーンショットをそのまま渡せます。

ファイルパス指定

画像ファイルのパスをプロンプトに含めます。@screenshot.png のように指定します。

Bashモード（!プレフィックス）

! で始めると、Claudeを経由せずにシェルコマンドを直接実行します。

! git log --oneline -5
直近5件のコミットログが即座に表示される

使い分けのポイント

単純なコマンド実行は ! で直接実行した方が高速です。Claudeに依頼すると、ツール呼び出しのオーバーヘッドが発生します。一方、「このログを分析して問題を見つけて」のように判断が必要な場合はClaudeに任せましょう。

パイプ入力

標準入力からデータをClaude Codeに渡すことができます。他のコマンドの出力をそのまま分析に使えます。

cat error.log | claude -p "このエラーの原因を特定して"
git diff HEAD~3 | claude -p "この変更のレビューをして"

タブ補完とプロンプトサジェスチョン

入力中にTabキーを押すと、ファイルパスやコマンドの補完候補が表示されます。また、セッション開始時にはプロジェクトに基づいたプロンプト候補が提案されることがあります。

パーミッションモード

参照: Permissions

Claude Codeはあなたのローカルマシンでツールを実行します。パーミッションシステムにより、どの操作に承認が必要かを制御できます。

パーミッションの基本

操作は大きく3つのカテゴリに分かれ、それぞれ承認の要否が異なります。

読み取り専用

ファイルの読み取り、Grep、Glob検索など。承認不要で自動実行されます。

Bash実行

シェルコマンドの実行。デフォルトでは毎回承認が必要です。

ファイル変更

ファイルの編集・作成・削除。デフォルトでは毎回承認が必要です。

パーミッションモード一覧

Shift+Tab で順番に切り替えられます。

D
default
標準モード。読み取りは自動、Bash実行とファイル変更は都度承認が必要です。最も安全なモードで、初心者に推奨されます。
E
acceptEdits
ファイル編集を自動承認します。Bash実行のみ承認が必要。コード修正を連続で行う作業に適しています。
P
plan
読み取り専用モード。ファイルの変更やコマンド実行は一切行いません。コードの分析や実装計画の立案に最適です。
A
dontAsk
全ての操作を自動承認します。承認プロンプトが一切表示されません。信頼できるタスクを効率的に実行する場合に使いますが、注意が必要です。
!
bypassPermissions
サンドボックスも含む全ての制限を無効化します。CLIフラグ --dangerously-skip-permissions で有効にします。CI/CD環境など特殊な用途向けです。

/permissions コマンド

/permissions を実行すると、現在許可されているツールの一覧と設定を確認できます。ここからルールの追加・削除も行えます。

パーミッションルールの構文

設定ファイルでツールごとに詳細なルールを定義できます。

// settings.json のパーミッション設定例
{
"permissions": {
"allow": [
"Read", // Readツール全体を許可
"Bash(git *)", // gitコマンドのみ許可
"Bash(npm test)", // 特定コマンドを許可
"Edit(src/**)", // srcディレクトリ内の編集を許可
"WebFetch(https://docs.example.com/*)", // 特定ドメインのみ許可
"mcp__serverName__toolName" // MCP特定ツールを許可
],
"deny": [
"Bash(rm -rf *)" // 危険なコマンドを拒否
]
}
}

ルール記法のパターン

  • Tool -- ツール全体を対象
  • Tool(specifier) -- 特定の引数パターンを対象
  • * -- ワイルドカード（任意の文字列にマッチ）
  • Agent -- サブエージェントの起動を対象

設定の優先順位

パーミッション設定は複数の場所で定義でき、以下の優先順位で適用されます（上が最優先）。

Managed（組織管理）
最も優先度が高い
↓
CLI フラグ
起動時の引数
↓
Local（.claude/settings.local.json）
gitignoreされる個人設定
↓
Project（.claude/settings.json）
リポジトリにコミットされるチーム共有設定
↓
User（~/.claude/settings.json）
全プロジェクト共通の個人設定

Plan Mode

参照: Common Workflows

Plan Modeは読み取り専用の分析モードです。コードを変更せずに調査・分析・計画立案を行います。実装前に「まず考える」フェーズを設けることで、手戻りを大幅に減らせます。

Plan Modeとは

Plan Modeでは、Claude Codeはファイルの読み取りとコード検索のみ行い、一切のファイル変更やコマンド実行を行いません。コードベースを安全に探索し、実装計画を立てるためのモードです。

いつ使うべきか

マルチステップの実装

大きな機能追加やリファクタリングの前に、影響範囲を調査し、実装ステップを整理する場合。

コード探索

新しいコードベースを理解したい場合。アーキテクチャや依存関係を把握できます。

インタラクティブな設計

Claudeと対話しながら設計を練る場合。安全に議論を進められます。

有効化の方法

1
Shift+Tab で切り替え
セッション中にパーミッションモードを順番に切り替えて「plan」を選択します。最も手軽な方法です。
2
CLIフラグで指定
claude --permission-mode plan
3
パイプモードで使用
claude -p --permission-mode plan "このコードの改善点を分析して"

推奨ワークフロー（4フェーズ）

1
Explore（探索）
Plan Modeでコードベースを探索。関連ファイル、依存関係、既存の実装パターンを把握します。
↓
2
Plan（計画）
探索結果をもとに、具体的な実装ステップを立案。変更対象のファイル、変更内容、テスト方針を明確にします。
↓
3
Implement（実装）
Shift+TabでdefaultまたはacceptEditsモードに切り替え、計画に沿って実装を進めます。
↓
4
Commit（コミット）
変更をレビューし、コミットメッセージを生成してコミットします。

デフォルトに設定する

// settings.json でPlan Modeをデフォルトにする
{
"permissions": {
"default_mode": "plan"
}
}

モデル設定

参照: Model Config / Fast Mode

Claude Codeでは使用するモデルや推論パラメータを柔軟に設定できます。タスクの複雑さや速度要件に応じて最適な設定を選びましょう。

モデルの選択

以下の3つの方法でモデルを変更できます。

/model

セッション中にスラッシュコマンドで変更。ピッカーが表示されます。

--model フラグ

起動時にモデルを指定します。
claude --model opus

利用可能なモデル

Opus

最も高性能なモデル。複雑なアーキテクチャ設計、大規模リファクタリング、難しいバグの調査に最適です。レスポンスは遅めですが、品質は最高です。

Sonnet

バランスの取れたデフォルトモデル。日常的なコーディング、コードレビュー、テスト作成など幅広い用途に対応します。速度と品質のバランスが優れています。

Haiku

最も高速なモデル。簡単な質問、コードの説明、小さな修正など、速度が重要な場面で使います。コスト効率も最も良いです。

Effortレベル（推論の深さ）

モデルがどの程度深く「考える」かを制御します。

low

素早い回答。単純な質問向け。

medium

標準的な推論。一般的なタスク向け。

high

深い推論。複雑な問題向け。デフォルト。

max

最大限の推論。最も難しい問題向け。

claude --effort max
最大の推論深度で起動。またはセッション中に /effort max で変更。

拡張思考（Extended Thinking）

拡張思考はデフォルトで有効になっています。有効にすると、モデルが回答を生成する前に「内部で考える」プロセスが追加され、複雑な問題での回答品質が向上します。

"ultrathink" キーワード

プロンプトに「ultrathink」というキーワードを含めると、拡張思考の深さが最大化されます。特に難解な問題や多段階の推論が必要な場合に試してみてください。

高速モード（Fast Mode）

/fast コマンドで高速モードに切り替えると、レスポンス速度を優先した設定になります。簡単な質問や素早い操作が必要な場面で活用してください。

復習チェック

次に進む前に、以下を確認してみましょう：

  • claude、claude "query"、claude -p "query" の違いを説明できる
  • Shift+Tab で切り替わるパーミッションモードを列挙できる
  • Plan Modeの4フェーズワークフローを説明できる
  • /compact と /clear の違いを理解している
  • タスクの複雑さに応じたモデル選択ができる
  • @、!、/ の各接頭辞の役割を理解している
