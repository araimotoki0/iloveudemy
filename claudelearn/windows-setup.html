注意

Claude Codeを使うにはClaudeの有料プラン（Pro / Max）が必要です。 無料プランでは使用できません。

本講座では WSL を使います

Claude Code は PowerShell やコマンドプロンプトからも実行できます。 ただし本講座では WSL（Windows Subsystem for Linux） を使う方法で進めます。

WSL とは

WSL（Windows Subsystem for Linux）は、Windows の中で Linux 環境を動かせる Microsoft 公式の仕組みです。

そもそも、パソコンの OS は大きく分けて Windows・macOS・Linux の3つがあります。 このうち macOS と Linux は同じ Unix 系の設計をベースにしていて、ターミナルのコマンド体系がほぼ同じです。 WSL を使うと、Windows の中に Linux 環境が入るので、Mac ユーザーと同じコマンドが使えるようになります。

Linux は開発の世界では標準

普段使うパソコンは Windows が多いですが、開発の世界では事情が違います。 Webサービスを動かすサーバーの大半は Linux です（世界のWebサーバーの約8割が Linux で稼働しています）。

npm、Docker、Git などの開発ツールも Linux / Unix を前提に作られています。 Mac はそのまま Unix 系なので問題ありませんが、Windows で開発するなら WSL で Linux 環境を用意するのが現在の標準的なやり方です。 実際、Microsoft 自身が WSL を開発して推進しています。

WSL 2 で実用的になった

WSL には初期の WSL 1 と現行の WSL 2 があります。 WSL 1 は Linux コマンドの互換レイヤーでしたが、WSL 2 では本物の Linux カーネルが動いており、 完全な Linux 環境として使えます。 ファイルシステムの速度も大幅に改善され、Docker もネイティブに動きます。 現在 wsl --install で入るのは WSL 2 です。

なぜ本講座で WSL を使うのか

  • 講座の内容がそのまま動く — 本講座は Mac のターミナルで操作を見せます。WSL なら同じ Linux コマンド体系なので、講座で見せるコマンドをそのままコピペして実行できます。PowerShell だとコマンドの書き方が違う場面が頻繁に出てきます
  • Claude Code が内部で使うのも Linux コマンド — Claude Code がファイル操作や検索を行うとき、内部的に ls, grep, cat などの Linux コマンドを実行します。WSL ならこれらがネイティブに動きます。PowerShell で使う場合は別途 Git for Windows のインストールが必要です
  • サンドボックスが使える — Claude Code には、コマンド実行を安全に隔離するサンドボックス機能があります。これは WSL 2 でのみ利用可能で、PowerShell ネイティブ環境では使えません

PowerShell でも使えなくはない

Claude Code は PowerShell / CMD / Git Bash でも動作します。 Git for Windows をインストールすれば、WSL なしでも使えます。 ただし、講座の進行と合わせるなら WSL の方がスムーズです。

VS Code のインストール

Visual Studio Code（VS Code）は、Microsoftが無料で提供しているコードエディタです。 Claude Codeはターミナルで動くツールですが、VS Codeにはターミナルが内蔵されているので、 コードを見ながらClaude Codeを操作できます。

他のエディタでもOK

Claude Codeはエディタに依存しません。JetBrains（IntelliJ, WebStormなど）、Cursor、サクラエディタ、秀丸など、 使い慣れたエディタがあればそちらで構いません。本講座ではVS Codeを使って説明しますが、 Claude Code自体はターミナルさえあれば動きます。 なお、エディタのショートカットキーは今の開発では重要です。AIがコードを書く時代になっても、コードを読む・移動する・選択する操作は人間の仕事なので、覚えておいて損はありません。

インストール手順

  1. 公式サイト（https://code.visualstudio.com/）にアクセス
  2. 「Download for Windows」をクリック
  3. ダウンロードした .exe ファイルを実行
  4. インストーラーの指示に従って進める（デフォルト設定でOK）
  5. 「PATHへの追加」にチェックが入っていることを確認（デフォルトでオン）

WSL拡張機能を入れよう

VS Code をインストールしたら、拡張機能「WSL」（Microsoft公式）を入れてください。 これを入れると、VS Code から WSL 内のファイルを直接開いて編集できます。 VS Code の拡張機能タブ（Ctrl + Shift + X）で「WSL」と検索してインストールします。

WSL（Windows Subsystem for Linux）のインストール

WSL は Windows の中で Linux を動かす Microsoft 公式の仕組みです。 コマンド1つでインストールできます。

システム要件

  • Windows 10 バージョン 2004 以降、または Windows 11
  • 「仮想マシン プラットフォーム」が有効になっていること（WSL インストール時に自動で有効化されます）

インストール手順

PowerShell を管理者として開き、以下を実行します。

（スタートメニューで「PowerShell」を検索 → 右クリック → 「管理者として実行」）

wsl --install
Ubuntu がデフォルトでインストールされます。完了後、PCを再起動してください。

初回セットアップ

再起動後、自動的に Ubuntu のウィンドウが開きます（開かない場合はスタートメニューから「Ubuntu」を起動）。 ユーザー名とパスワードの設定を求められます。

  • ユーザー名 — 英字の短い名前でOK（例: motoki）
  • パスワード — 入力時に画面に表示されませんが、ちゃんと入力されています。忘れないようにしてください

動作確認

PowerShell または コマンドプロンプトで以下を実行します。

wsl --version
WSL バージョン: 2.x.x.x と表示されれば成功です。

WSL に入るには

PowerShell やコマンドプロンプトで wsl と入力するだけで Linux 環境に入れます。 スタートメニューから「Ubuntu」を起動しても同じです。 VS Code で WSL 拡張機能を使っている場合は、左下の「><」アイコンから「WSL への接続」を選ぶと、 VS Code のターミナルが WSL 内で動きます。

VS Code から WSL に接続する方法

VS Code で Ctrl + Shift + P → 「WSL: Connect to WSL」を選択するか、 左下の「><」アイコンをクリックして「WSL への接続」を選びます。 接続すると、VS Code のターミナルが WSL 内の bash/zsh になり、ファイルツリーも WSL 側のファイルが表示されます。

Claude Code のインストール

Claude Codeのインストールはたった1行のコマンドで完了します。 ネイティブインストーラーが提供されており、Node.jsなどの依存関係は不要です。

必ず WSL 内で実行してください

以下のコマンドは Windows の PowerShell やコマンドプロンプトではなく、 WSL の Ubuntu ターミナルで実行してください。 wsl と入力して Linux 環境に入った状態で実行します。

インストールコマンド

WSL のターミナルで以下を実行します。

curl -fsSL https://claude.ai/install.sh | bash

PATHを通す

インストール後に claude: command not found と出る場合は、 PATHの設定が必要です。以下を実行してください。

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc
~/.bashrc にPATH設定を追加して、即座に反映します。

インストール確認

claude --version

Claude Code を起動して連携する

ターミナルで claude と入力して起動します。

claude

初回起動時にブラウザが開き、先ほど作成した有料アカウントとの連携を求められます。 ログインして許可すれば完了です。

ブラウザが自動で開かない場合

WSL からブラウザが自動で開かないことがあります。 その場合、ターミナルに表示される URL をコピーして、Windows 側のブラウザに貼り付けてアクセスしてください。

自動アップデート

ネイティブインストーラーで入れたClaude Codeは、起動時に自動でアップデートを確認します。手動で更新する必要はありません。

おすすめツール（オプション）

必須ではありませんが、開発作業の効率が上がるツールを2つ紹介します。

Windows Terminal — モダンなターミナル

Microsoft 公式のターミナルアプリです。タブで PowerShell・コマンドプロンプト・WSL を切り替えられます。 画面分割にも対応しており、WSL を使うなら入れておくと便利です。 Windows 11 には標準搭載されています。

  1. Microsoft Store で「Windows Terminal」を検索してインストール
  2. 起動してタブの「＋」ボタンから「Ubuntu」を選ぶと WSL に入れる

よく使うショートカット

Alt + Shift + D 画面分割 / Ctrl + Shift + T 新しいタブ / Ctrl + Shift + W タブを閉じる

PowerToys — ウィンドウ管理ほか

Microsoft 公式のユーティリティ集です。ウィンドウのスナップ配置（FancyZones）、 クリップボード履歴、ファイル名の一括変更など、開発に便利な機能がまとまっています。

  1. Microsoft Store で「PowerToys」を検索してインストール
  2. 起動して使いたい機能を有効化する

特に便利な機能

FancyZones — ウィンドウを画面の左半分・右半分にドラッグするだけで配置。カスタムレイアウトも作れる
PowerToys Run — Alt + Space でランチャーを起動。アプリやファイルを素早く検索
