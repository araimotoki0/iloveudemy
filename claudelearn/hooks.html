Hooksとは何か ― AIの作業に「自動処理」を差し込む

参照: Hooks - 公式ドキュメント / Hooks ガイド

Hooksとは、AIの作業の特定タイミングで自動的に何かを実行する仕組みです。

日常生活で例えると

料理が完成したらベルを鳴らす
トリガー: 料理の完成
アクション: ベルを鳴らす
→ Claude Codeでは「作業完了したら通知を出す」
包丁を使う前に手を洗う
トリガー: 包丁を使う前
アクション: 手を洗う
→ Claude Codeでは「コミット前にリントを実行する」

Gitのpre-commitフックに馴染みがあれば

Gitでは pre-commit フックを使って、コミット前にリントやテストを自動実行できます。 Claude CodeのHooksはそれと同じ考え方で、AIの各種アクションをトリガーにして任意のコマンドやHTTPリクエストを自動実行できます。

Hooksの全体像

Hooksの仕組み
1. イベントが発生する
例: AIがツールを使おうとした、セッションが始まった、作業が完了した
↓
2. matcherで条件を判定（オプション）
例: 「Bashツールで、git commit* というコマンドの場合だけ」
↓
3. 登録されたコマンド/HTTPリクエストを自動実行
例: npm run lint を実行、macOS通知を表示、Slackに投稿

Hooksの設定場所

Hooksは settings.json に記述します。
/hooks コマンドでGUIから設定することもできますし、直接JSONファイルを編集することもできます。

使えるイベント一覧

Hooksで使えるイベント（トリガー）は以下の通りです。それぞれのイベントで「いつ発火するか」と「どんな使い方ができるか」をまとめました。

イベント名 発火タイミング 具体的な使い道
PreToolUse ツール実行の前 ・特定のファイルの編集をブロック
・コミット前にリントを実行
・危険なコマンドの実行を阻止
PostToolUse ツール実行の後 ・コミット後に自動でリント
・ファイル編集後にフォーマッタを実行
・ビルド結果のログ記録
SessionStart セッション開始時 ・作業環境のセットアップ
・依存パッケージの確認
・ブランチ情報の取得
Stop AIが作業を完了した時 ・macOS/Linux通知で完了を知らせる
・Slackに完了メッセージを投稿
・作業ログの記録
PermissionRequest パーミッション確認時 ・通知音を鳴らして注意を促す
・ユーザーの操作待ちであることを通知

PreToolUseとPostToolUseが最も使う場面が多い

ツールの実行前後にフックを差し込むことで、コード品質の維持や自動フォーマットなど、開発ワークフローを大幅に効率化できます。 matcher と組み合わせることで、特定のツール・特定のコマンドにだけ反応させることも可能です。

イベント発火の流れ

SessionStart → セッション開始時に1回だけ発火 ↓ AIがタスクを処理中... ↓ PreToolUse → ツール実行の直前（matcherで絞り込み可能） ↓ ツール実行（Bash, Edit, Read など） ↓ PostToolUse → ツール実行の直後（matcherで絞り込み可能） ↓ （繰り返し...） ↓ Stop → AIが全作業を完了した時に発火

ハンズオン: 「作業完了したら通知する」Hookを作ろう

最初のハンズオンとして、AIの作業が完了したらmacOSの通知を表示するHookを作ります。 長いタスクを任せている間に別の作業をしていても、完了を見逃しません。

1設定ファイルを開く

Claude Codeで /hooks コマンドを実行するか、直接 settings.json を編集します。

/hooks
GUIでHooksの設定画面が開きます

または、直接JSONファイルを編集する場合は以下のパスです。

code ~/.claude/settings.json
プロジェクト固有の場合: .claude/settings.json

2settings.jsonにHookを追加

以下の設定を settings.json に追加します。

// settings.json { "hooks": { "Stop": [ { "hooks": [ { "type": "command", "command": "osascript -e 'display notification \"Claudeの作業が完了しました\" with title \"Claude Code\"'" } ] } ] } }

3設定内容の解説

"Stop"
イベント名。AIが作業を完了した時に発火する
"type": "command"
シェルコマンドを実行するタイプ（他に "http" もある）
"command": "osascript -e ..."
macOSの osascript コマンドで通知を表示する。Linuxの場合は notify-send を使う

4動作を確認する

Claude Codeで何かタスクを実行してみましょう。AIが作業を完了すると、macOSの通知センターに通知が表示されます。

claude "Hello と表示するだけのPythonスクリプトを作って"
→ AIが作業完了 → macOS通知が表示される!

Linux/WSLの場合

osascript はmacOS専用です。Linuxでは代わりに以下を使います:

"command": "notify-send 'Claude Code' 'Claudeの作業が完了しました'"

音も鳴らしたい場合

macOSの場合、afplay コマンドでシステムサウンドを再生できます:

"command": "osascript -e 'display notification \"完了\" with title \"Claude Code\"' && afplay /System/Library/Sounds/Glass.aiff"

ハンズオン: 「コミット前に自動リント」Hookを作ろう

次のハンズオンでは、AIが git commit を実行した後に自動でリントを走らせるHookを作ります。 AIがコミットした後にコード品質をチェックし、問題があればすぐに検知できます。

1やりたいこと

AIが git commit -m "..." を実行
↓ PostToolUseイベント発火
matcherで「Bashツール」かつ「git commit*」に一致するか判定
↓ 一致した場合のみ
npm run lint が自動で実行される

2settings.jsonに追加する設定

// settings.json { "hooks": { "PostToolUse": [ { "matcher": { "tool_name": "Bash", "command": "git commit*" }, "hooks": [ { "type": "command", "command": "npm run lint" } ] } ] } }

3設定内容の解説

"PostToolUse"
ツール実行の「後」に発火するイベント
"matcher" オブジェクト
条件を絞り込む。tool_name でツール名、command でコマンドパターン（ワイルドカード * 使用可）を指定
"command": "npm run lint"
条件に一致した時に実行されるコマンド

4動作確認

Claude Codeにコードを書かせてコミットまでお願いしてみましょう。

claude "utils.jsにformatDate関数を追加してコミットして"
→ AIがコードを書く → AIが git commit を実行 → 自動で npm run lint が走る! → リントエラーがあればAIに結果が伝わる

npm run lint が設定されていない場合

プロジェクトに package.json の scripts.lint が定義されていない場合はエラーになります。 ESLint等のリンターがセットアップ済みのプロジェクトで試してください。

応用: フォーマッタも自動実行

同じ仕組みで、ファイル編集後に prettier を自動実行することもできます:

{ "PostToolUse": [ { "matcher": { "tool_name": "Edit" }, "hooks": [ { "type": "command", "command": "npx prettier --write $CLAUDE_FILE_PATH" } ] } ] }

matcherで条件を絞る ― 必要な時だけHookを発火させる

matcher を使うと、特定の条件に一致した時だけHookを実行できます。 matcherを指定しない場合、そのイベントのすべてのケースでHookが発火します。

ツール名でフィルタリング

tool_name を指定すると、特定のツールが使われた時だけHookが発火します。

Bash
シェルコマンドの実行時に発火
Edit
ファイル編集時に発火
Write
ファイル作成時に発火
Read
ファイル読み取り時に発火
Glob
ファイル検索時に発火
Grep
テキスト検索時に発火

コマンドパターンでフィルタリング

command を指定すると、Bashツールの場合に実行されるコマンドをワイルドカードで絞り込めます。

// git commit で始まるコマンドにだけ反応 "matcher": { "tool_name": "Bash", "command": "git commit*" } // npm test で始まるコマンドにだけ反応 "matcher": { "tool_name": "Bash", "command": "npm test*" } // docker で始まるコマンドにだけ反応 "matcher": { "tool_name": "Bash", "command": "docker*" }

matcherなし vs matcherあり の違い

matcherなし
すべてのツール実行で発火
→ Read, Edit, Bash, Grep... 全部で発火してしまう
パフォーマンスに影響する可能性あり
matcherあり
条件に一致した時だけ発火
→ 必要な場面だけで実行される
効率的で予測可能

ハンズオン: 特定ファイルの編集をブロックする

PreToolUse と matcher を組み合わせて、特定のファイルをAIに編集させないようにすることもできます。 Hookのコマンドが非ゼロの終了コードを返すと、そのツール実行がブロックされます。

{ "PreToolUse": [ { "matcher": { "tool_name": "Edit" }, "hooks": [ { "type": "command", "command": "echo $CLAUDE_FILE_PATH | grep -q 'package-lock.json' && exit 1 || exit 0" } ] } ] }

この例では、AIが package-lock.json を編集しようとした場合、Hookが終了コード1を返してブロックします。

HTTP Hooks ― 外部サービスと連携する

Hooksはシェルコマンドだけでなく、HTTP POSTリクエストを送ることもできます。 これにより、Slack、Discord、Webhookなど外部サービスとの連携が可能になります。

HTTP Hookの基本構文

{ "hooks": { "Stop": [ { "hooks": [ { "type": "http", "url": "https://hooks.slack.com/services/T.../B.../xxx", "headers": { "Content-Type": "application/json" }, "body": "{\"text\": \"Claude Codeの作業が完了しました\"}" } ] } ] } }

Slack通知の例（概念）

Slackの Incoming Webhook URLを取得すれば、AIの作業完了をチームのSlackチャンネルに通知できます。

1. Slack AppでIncoming Webhookを作成
Slack APIの設定画面で、通知を送りたいチャンネルにWebhookを設定
2. Webhook URLをHookのurlに設定
https://hooks.slack.com/services/T.../B.../xxx のようなURLをコピー
3. AIが作業完了 → Slackに自動投稿
チームメンバーがリアルタイムで作業完了を確認できる

Webhook URLの管理に注意

Webhook URLは秘密情報です。settings.json をGitにコミットする場合、URLが公開されないよう注意してください。 環境変数を使うか、ユーザーレベルの ~/.claude/settings.json に記述するのが安全です。

commandタイプでもHTTPリクエストは送れる

type: "command" で curl コマンドを使えば、同じことが実現できます。 type: "http" はより宣言的で、コマンドの書き方を覚える必要がないというメリットがあります。

// commandタイプでcurlを使う場合 { "type": "command", "command": "curl -X POST -H 'Content-Type: application/json' -d '{\"text\":\"完了\"}' https://hooks.slack.com/..." }

まとめ: どのタイプを使う？

type: "command"
・ローカルのコマンド実行に最適
・リント、フォーマット、通知音、OS通知
・柔軟性が高い（パイプ、条件分岐など）
type: "http"
・外部サービスとの連携に最適
・Slack、Discord、Webhook
・宣言的でシンプル
