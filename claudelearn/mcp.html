MCPとは何か ― AIと外部ツールをつなぐ「USBポート」

参照: Claude Code公式ドキュメント - MCP

MCP（Model Context Protocol）は、Claude Codeに外部ツールを接続するための標準規格です。
パソコンのUSBポートに例えるとわかりやすいでしょう。

MCPは「USBポート」のようなもの

🤖
Claude Code
（パソコン本体）
🔌
MCP
（USBポート＝接続規格）
🌐
外部ツール
GitHub / Slack / DB / API...
USBポートがあれば、マウスもキーボードもプリンターも接続できる。
MCPがあれば、GitHubもSlackもデータベースもClaude Codeに接続できる。

MCPがない世界 vs MCPがある世界

MCPなし

1. ブラウザでGitHubを開く
2. Issueの内容を手動でコピー
3. Claude Codeに貼り付けて質問
4. AIの回答を手動でGitHubに反映

毎回この手作業が必要...

MCPあり

1. Claude Codeに「Issue #42を修正して」と言うだけ
2. AIが自動でIssueを読む
3. AIがコードを修正
4. AIがPRを作成

全部AIが直接やってくれる!

技術的に言うと

MCPサーバーはローカルで動く小さなプログラムです。Claude Codeが「GitHub上のIssue一覧を取得して」と依頼すると、MCPサーバーがGitHub APIを叩いて結果を返します。
Claude Code自体が直接外部APIにアクセスするのではなく、MCPサーバーが橋渡し役をします。

MCPの通信フロー

// あなたの指示 "GitHubのIssue #42を見て、修正PRを作って" | v [Claude Code] --MCP Protocol--> [GitHub MCPサーバー] --API--> [GitHub] (ローカルで動作) // MCPサーバーが提供する「ツール」の例: - list_issues() // Issue一覧を取得 - get_issue(id) // 特定のIssueを取得 - create_pr(title) // PRを作成 - add_comment(text) // コメントを追加

ハンズオン ― MCPサーバーを接続してみよう

参照: Claude Code公式ドキュメント - MCP

実際にMCPサーバーを設定して、Claude Codeの機能を拡張してみましょう。
まずは一番シンプルなFilesystem MCPサーバーから始めます。

方法1: /mcp コマンドで確認する

まず、現在接続されているMCPサーバーを確認してみましょう。

/mcp
Claude Code内で /mcp と入力すると、現在のMCPサーバーの状態が表示されます。
まだ何も設定していなければ「No MCP servers configured」と出ます。

ハンズオン: まず試してみよう

Claude Codeを起動して、/mcp と入力してみてください。
何が表示されるか確認しましょう。何も設定していなければ空のリストが返ってきます。

方法2: settings.jsonで手動設定する

MCPサーバーの設定はsettings.jsonに記述します。
設定ファイルの場所は3つのスコープがあります。

ユーザー（グローバル）
~/.claude/settings.json

全プロジェクトで使える

プロジェクト（共有）
.claude/settings.json

チームで共有される

プロジェクト（個人）
.claude/settings.local.json

自分だけの設定

どこに書くべき?

迷ったら ~/.claude/settings.json（ユーザースコープ）に書きましょう。
全プロジェクトで使えて、個人の環境に閉じるので安全です。

ハンズオン: Filesystem MCPサーバーを設定してみよう

Filesystem MCPサーバーを使うと、Claude Codeが指定したディレクトリ内のファイルを安全に読み書きできるようになります。
以下の手順で設定してみましょう。

1
settings.jsonを開く（または作成する）
cat ~/.claude/settings.json
ファイルが存在しない場合は、新規作成します。
既にある場合は中身を確認しましょう。
2
MCPサーバーの設定を追加する

以下の内容を ~/.claude/settings.json に記述します。
/path/to/allowed/dir は実際にアクセスを許可したいディレクトリに置き換えてください。

// ~/.claude/settings.json { "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-filesystem", "/path/to/allowed/dir"] } } }

設定の読み方:
"filesystem" ... MCPサーバーの名前（自由につけられる）
"command" ... 起動コマンド（npxでnpmパッケージを実行）
"args" ... コマンドの引数（パッケージ名とアクセス許可ディレクトリ）
"-y" ... npxの確認プロンプトを自動でYes

3
Claude Codeを再起動して確認する

設定後、Claude Codeを再起動（またはリロード）して、/mcp で確認します。

/mcp
MCP Servers:
  filesystem: connected

このように「connected」と表示されれば成功です!
4
実際に使ってみる

接続できたら、Claude Codeに話しかけてみましょう。

指定したディレクトリにあるファイルの一覧を教えて
Claude CodeがFilesystem MCPサーバーを使って、
指定ディレクトリ内のファイル一覧を取得して表示してくれます。

ここまでのまとめ

MCPサーバーの接続は、たった2ステップです。
1. settings.jsonに設定を書く
2. Claude Codeを再起動する

これだけで、Claude Codeに新しい能力が追加されます。

人気のMCPサーバー ― まず入れるならこれ

参照: Claude Code公式ドキュメント - MCP / 公式MCPサーバー一覧

MCPサーバーは多数公開されています。
ここでは特に開発者に人気の定番サーバーを紹介します。

🐙
GitHub MCP
Issue、PR、リポジトリの操作ができます。
「Issue #42を修正してPRを作って」のような開発フロー全体をAIに任せることが可能に。
"github": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-github"], "env": { "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx" } }

GITHUB_TOKENが必要です。GitHubの Settings > Developer settings > Personal access tokens で発行してください。

📁
Filesystem MCP
指定したディレクトリ内のファイルを読み書きできます。
Claude Codeの標準機能と似ていますが、プロジェクト外のファイルにもアクセスできるのが特徴。
"filesystem": { "command": "npx", "args": [ "-y", "@anthropic-ai/mcp-filesystem", "/Users/me/documents" ] }

複数のディレクトリを許可したい場合は、argsにパスを追加するだけです。

🌐
Fetch MCP
Web上の情報を取得できます。
「このURLのページ内容を要約して」「APIのレスポンスを確認して」のようなWebアクセスが可能に。
"fetch": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-fetch"] }

追加設定なしですぐ使えます。URLを渡すだけでWebページの内容を取得できます。

🔧
その他の人気サーバー
Slack MCP ... チャンネル閲覧、メッセージ送信
PostgreSQL MCP ... DB操作、クエリ実行
Puppeteer MCP ... ブラウザ自動操作
Memory MCP ... 長期記憶の保存

ハンズオン: 複数のMCPサーバーを同時に設定する

MCPサーバーは複数同時に使えます。以下は実践的な設定例です。

// ~/.claude/settings.json の例（複数サーバー） { "mcpServers": { "github": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-github"], "env": { "GITHUB_TOKEN": "ghp_xxxxxxxxxxxx" } }, "filesystem": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-filesystem", "/Users/me/documents"] }, "fetch": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-fetch"] } } }

やってみよう

上の設定を参考に、自分のsettings.jsonに好きなMCPサーバーを追加して、/mcp で確認してみましょう。
まずは設定不要のFetch MCPから試すのがおすすめです。

MCPサーバーの探し方・入れ方

参照: Claude Code公式ドキュメント - MCP

「どんなMCPサーバーがあるの?」「どうやって入れるの?」
3つの方法を紹介します。

方法1: 対話式でインストールする（初心者におすすめ）

Claude Codeの中で/mcpコマンドを使うと、対話的にMCPサーバーを追加できます。

1
Claude Code内で /mcp コマンドを実行
/mcp
MCPサーバーの管理メニューが表示されます。
「Add」を選択して、新しいサーバーを追加できます。
2
サーバー名とコマンドを入力

対話形式で以下を聞かれます:

Server name: fetch
Server command: npx -y @anthropic-ai/mcp-fetch
Scope: User
3
自動で settings.json に追加される

手動でJSONを編集する必要がありません。初心者に最もおすすめの方法です。

方法2: settings.jsonを直接編集する

前のセクションで紹介した方法です。JSONの書き方がわかっていれば最も柔軟な方法です。
環境変数（env）の設定が必要なサーバーは、この方法が確実です。

// 環境変数が必要な場合の設定例 { "mcpServers": { "slack": { "command": "npx", "args": ["-y", "@anthropic-ai/mcp-slack"], "env": { "SLACK_BOT_TOKEN": "xoxb-xxxxxxxxxxxx", "SLACK_TEAM_ID": "T0123456789" } } } }

セキュリティの注意点

参照: Claude Code公式ドキュメント - MCP

MCPは便利ですが、セキュリティを意識して使うことが重要です。
以下の3つのポイントを覚えておきましょう。

1
MCPサーバーはローカルで動くプロセス

MCPサーバーはあなたのPC上で動作するプログラムです。
つまり、そのプログラムはあなたのPCの権限でファイルやネットワークにアクセスできます。
悪意のあるMCPサーバーを入れると、ファイルを読まれたり、データを送信されたりする可能性があります。

2
信頼できるソースからのみインストール

MCPサーバーは以下の信頼できるソースから入れましょう。

✅
Anthropic公式
@anthropic-ai/ で始まるパッケージ
✅
公式リポジトリ
modelcontextprotocol/ のGitHub
⚠️
コミュニティ製
スター数やREADMEを確認
3
パーミッション設定との関係

MCPサーバーが提供するツールも、Claude Codeのパーミッションシステムの対象です。
MCPサーバー経由の操作であっても、初回はClaude Codeが「このツールを使っていいですか?」と確認してきます。

// MCPツールの使用時にパーミッション確認が出る Claude wants to use mcp_github_create_pr Title: "Fix issue #42" Base: "main" Allow? [y/n/always]

always を選ぶと、次回から同じツールの確認がスキップされます。
信頼できるMCPサーバーのツールだけ always にしましょう。

セキュリティチェックリスト

MCPサーバーを追加する前に確認しましょう:

1. そのパッケージは公式/信頼できる開発者のものか?
2. GitHubリポジトリにソースコードが公開されているか?
3. 環境変数にトークンを設定する場合、そのトークンの権限は最小限か?
4. settings.jsonにトークンを直書きする場合、そのファイルはGitで管理されていないか?

やってはいけないこと

settings.jsonをGitにコミットしない ... APIトークンが漏洩します
知らないMCPサーバーをそのまま入れない ... まずソースコードを確認
全権限のトークンを使わない ... 必要最小限のスコープに絞る
