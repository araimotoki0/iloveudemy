まずは体験

コマンドを覚える前に、まずは「Claude Codeで何ができるか」を体験しましょう。
受託開発会社 株式会社システムソリューションズ の営業用プレゼン資料を、たった一言のプロンプトで作ります。

Claude Codeを起動する

ターミナルで作業用のフォルダに移動し、claude と打つだけ。

mkdir ~/nextcode-demo && cd ~/nextcode-demo
claude
Claude Code を起動。インタラクティブモードに入ります。

プレゼン資料を作らせる

起動したら、以下のように話しかけるだけです。

あなた
受託開発会社「株式会社システムソリューションズ」の営業用プレゼン資料をHTMLで作って。AWSを活用したインフラ構築・運用、クラウド移行支援、大規模システムのモダナイゼーションが得意な会社として、企業向けの営業プレゼンをモダンなデザインで作って。

Python のインストール（補足）

PowerPoint変換など、Pythonライブラリを使う場面もあります。バージョン管理ができる pyenv 経由がおすすめです。

Mac / Windows（WSL2）共通：

curl https://pyenv.run | bash

▼ Mac

echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.zshrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.zshrc
echo 'eval "$(pyenv init -)"' >> ~/.zshrc

▼ Windows（WSL2）

echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc

ターミナルを再起動した後：

pyenv install 3.12 && pyenv global 3.12

Vercelにデプロイする

作ったプレゼン資料をお客様にURLで共有してみましょう。

Node.js のインストール

Vercel CLIはnpm経由でインストールするため、まずNode.jsが必要です。バージョン管理ができる nvm 経由がおすすめです。

Mac / Windows（WSL2）共通：

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.4/install.sh | bash

ターミナルを再起動した後：

nvm install --lts

Vercel CLIのインストール

npm i -g vercel
vercel login

CLAUDE.md ― プロジェクトのルールをClaude Codeに伝える

参照: CLAUDE.md

仕組み

Claude Codeはセッション開始時にCLAUDE.mdをディスクから読み込み、メッセージ配列の先頭に注入します。以降のすべてのAPIリクエストに会話履歴の一部として含まれます。 つまり、あなたが「ログイン画面を作って」と打つと、実際にはこうなっています：

【自動で含まれる - ~/CLAUDE.md】 日本語で返答して 【自動で含まれる - プロジェクト/CLAUDE.md】 このプロジェクトはReact + TypeScript テストはVitest 【あなたが入力したプロンプト】 ログイン画面を作って

ユーザーは「ログイン画面を作って」としか打っていないのに、Claude Codeは日本語で、React + TypeScriptで、テストはVitestで書く。
セッション開始時に読み込まれたCLAUDE.mdの内容が、会話履歴の一部としてすべてのリクエストに含まれているからです。

なぜCLAUDE.mdが必要？

Claude Codeにはメモリ機能（会話から自動で学習・記憶）もありますが、メモリはClaude Codeが勝手に拾って保存するため、何が入っているか把握しづらくなりがちです。CLAUDE.mdは自分が意図的に書いたルールだけが入るため見通しが良く、確実です。

さらに、プロジェクトルートに置けばGitにコミットしてチーム全員で共有できます。これがCLAUDE.mdの本質です。

~/CLAUDE.md
  • 全プロジェクト共通の指示
  • ホームディレクトリに置く
  • 言語指定、作業スタイルなど
プロジェクト/CLAUDE.md
  • プロジェクト固有の指示
  • プロジェクトルートに置く
  • 技術スタック、コーディング規約など

まずはこれを設定しよう

ホームディレクトリに CLAUDE.md を作成して、日本語指定を書きます。

# ~/CLAUDE.md always respond in Japanese

たったこれだけで、Claude Codeは常に日本語で返答するようになります。

プロジェクトごとの指示も書ける

プロジェクトルートに置く CLAUDE.md には、そのプロジェクト固有のルールを書きます。

# CLAUDE.md（プロジェクトルート） ## 技術スタック - Next.js 15 (App Router) - TypeScript - Tailwind CSS ## ルール - テストは vitest を使用 - コミットメッセージは日本語で

これがClaude Codeの「記憶」

CLAUDE.mdに書いておけば、毎回同じ指示を出す必要がなくなります。チームメンバーとプロジェクトのCLAUDE.mdを共有すれば、全員が同じルールでClaude Codeを使えます。

セッションを続ける ― claude -c と claude -r

参照: Interactive Mode

先ほどのプレゼン作成、途中でターミナルを閉じてしまったらどうなるでしょう？
大丈夫。Claude Codeにはセッションを復帰させる方法があります。

claude -c ― 直前のセッションを続ける

「あ、さっきの作業の続きをやりたい」というとき。直前のセッションをそのまま再開します。

claude -c
直前のセッションの会話履歴をすべて引き継いで再開
あなた
さっきのプレゼンに「よくある質問」ページを追加して

Claudeは前の会話を覚えているので、「さっきの」で通じます。

claude -r ― 過去のセッション一覧から選ぶ

「3日前のあの作業に戻りたい」というとき。過去のセッション一覧が表示され、選んで再開できます。

claude -r
過去のセッション一覧が表示される → 選択して再開

使い分けのコツ

  • -c = Continue（直前のセッションを続ける）
  • -r = Resume（過去のセッション一覧から選んで復帰する）

ワンライナーで使う ― claude -p

参照: CLI Reference

インタラクティブモードに入らなくても、一発でClaude Codeに仕事をさせる方法があります。
claude -p でプロンプトを直接渡します。

基本の使い方

claude -p "このプロジェクトのREADMEを作って"
結果がターミナルに出力される（対話なし）

対話せず結果だけ欲しい場合に最適です。スクリプトやCI/CDにも組み込めます。

パイプで組み合わせる

他のコマンドの出力をClaude Codeに渡せます。これが非常に強力です。

cat src/app.js | claude -p "このコードをレビューして"
git diff | claude -p "この差分の変更内容を要約して"
cat error.log | claude -p "このエラーログの原因を分析して"

シェルの力 × AIの力

Unixのパイプ哲学とAIを組み合わせることで、ログ分析、コードレビュー、ドキュメント生成など、あらゆるテキスト処理を一行で実行できます。

ファイルを指定する ― @ マーク

Claude Codeに「このファイルを見て」と伝えたいとき、@ でファイルパスを指定できます。

基本の使い方

あなた
@src/app.js このファイルのバグを修正して

@ の後にファイルパスを書くと、そのファイルの内容がコンテキストとして渡されます。

複数ファイルも指定できる

あなた
@src/api.js @src/types.ts この2つのファイルの型を揃えて

URLも指定できる

あなた
@https://example.com/api/docs このAPIドキュメントを元にクライアントを作って

タブ補完が使える

@ を入力した後、Tab キーを押すとファイル名の候補が表示されます。パスを手打ちする必要はありません。

よく使うスラッシュコマンド

参照: Commands

インタラクティブモード中に / で始まるコマンドを入力すると、特別な操作ができます。
全部覚える必要はありません。よく使うものから順に紹介します。

1
/compact ― 会話を圧縮する
長い作業をしていると、会話履歴がどんどん長くなります。/compact を実行すると、それまでの会話を要約して圧縮し、コンテキストウィンドウの空きを作ります。
/compact
会話が要約され、コンテキストに空きができる
「動作が遅くなってきたな」と感じたら試してみてください。コンテキストが圧迫されている可能性があります。
2
/clear ― 会話をリセットする
「話題を完全に変えたい」「ゼロから始めたい」ときに使います。会話履歴をすべてクリアして、新しいセッションのようにリフレッシュします。
/clear
会話履歴がクリアされ、まっさらな状態に
3
/cost ― 使用量を確認する
現在のセッションで消費したトークン数やコストを表示します。「今月どれくらい使ったかな？」が気になるときに。
/cost
セッションのトークン使用量とコストが表示される
4
/model ― モデルを切り替える
セッション中にモデルを変更できます。「軽い作業はSonnet、複雑な設計はOpus」のように使い分けたいときに便利です。
/model
利用可能なモデル一覧が表示される → 選択して切り替え
Maxプランの場合、Opus / Sonnet / Haiku を自由に切り替えられます。
5
/login ― 再ログインする
セッションが切れたり、アカウントを切り替えたいときに使います。ブラウザが開いて認証フローが始まります。
/login
ブラウザが開き、Anthropicアカウントで再認証
6
/doctor ― 環境を診断する
「なんか動かない」「エラーが出る」というときの最初の一手。Claude Codeの環境設定、認証状態、依存関係をチェックして問題点を教えてくれます。
/doctor
環境の診断結果が表示される ― 問題があれば修正方法も提示

その他のコマンド

/ を入力するとオートコンプリートで一覧が出ます。全コマンドの詳細は「基本操作ガイド」ページにまとめてあります。

考えさせてから動かす ― Shift + Tab

参照: Common Workflows

Claude Codeにはコードを書かずに「まず考える」モードがあります。
Shift + Tab で Plan Mode に切り替わります。

通常モード（Agent Mode）
ファイルの読み書き・コマンド実行ができる
Shift + Tab
Plan Mode
読み取り専用 ― 調査・分析・計画のみ

どんな時に使うか

大きな変更の前に

「リファクタリングの方針を先に考えて」
→ 計画を確認してからAgent Modeに戻って実行

コードの理解

「このプロジェクトの構成を説明して」
→ 変更せず読み取りだけで安全に調査

バグの原因調査

「ログインできないバグの原因を調べて」
→ 原因を特定してから修正に移る

見積もり

「この機能を追加するのに必要な変更を洗い出して」
→ 影響範囲の事前把握

Plan Modeの注意点

Plan Modeではファイルの書き込みやコマンド実行ができません。計画が固まったら Shift + Tab で通常モードに戻して実行しましょう。

自分だけの設定 ― settings.local.json

参照: Settings

Claude Codeの設定は settings.json と settings.local.json の2種類があります。
チームで共有する設定と、自分だけの設定を分けられます。

.claude/settings.json
  • Gitで管理される（チーム共有）
  • プロジェクト共通の許可ルール
  • チームのコーディング規約など
.claude/settings.local.json
  • .gitignoreに追加推奨
  • 自分だけのカスタマイズ
  • 個人的なツール許可設定など

よく使う設定例

プロジェクトルートに .claude/settings.local.json を作成します。

// .claude/settings.local.json { "permissions": { "allow": [ "Bash(npm run *)", "Bash(git *)", "Read", "Write" ] } }

許可ルールで確認ダイアログを減らす

Claude Codeは安全のため、ファイル書き込みやコマンド実行時に確認を求めます。信頼できる操作を allow に追加しておくと、いちいち承認する手間が省けます。

グローバル設定もある

プロジェクト横断で適用したい設定は、ホームディレクトリに置きます。

// ~/.claude/settings.json（全プロジェクト共通） { "permissions": { "allow": [ "Bash(git status)", "Bash(git log *)", "Bash(git diff *)" ] } }

Bash(*) は非推奨

"Bash(*)" を設定するとすべてのコマンドが無条件で実行されます。セキュリティ上のリスクがあるため、必要なコマンドだけを個別に許可しましょう。

常に最新版を使う ― claude update

Claude Codeは頻繁にアップデートされます。新機能やバグ修正を逃さないために、定期的にアップデートしましょう。

claude update
最新バージョンに更新されます

バージョン確認

現在のバージョンは claude --version で確認できます。起動時に新しいバージョンがある場合は通知が表示されることもあります。

古いバージョンのまま使っていませんか？

Claude Codeは進化が速いツールです。「先週までなかった機能」が今週には使えることも珍しくありません。困ったらまず claude update を試してみてください。

画像を渡す ― スクリーンショットも読める

Claude Codeは画像も理解できます。デザインカンプ、エラー画面のスクリーンショット、ワイヤーフレームなど、画像を渡して指示を出せます。

画像の渡し方

ファイルパスで指定

@マークでパスを指定して画像を渡します。

あなた
@design.png このデザインをHTMLで再現して
コピー&ペースト

スクリーンショットをクリップボードにコピーして、そのままペーストできます。

あなた
[画像をペースト] このエラー画面を修正して

こんな使い方ができる

1
デザインカンプ → HTML
Figmaやデザインツールのスクリーンショットを渡して、HTMLに変換
2
エラー画面 → 修正
ブラウザのエラー画面をスクショして渡すだけで、原因を特定して修正
3
手書きメモ → 実装
ホワイトボードの写真やノートのスケッチから、仕様を読み取って実装

超重要

超重要
