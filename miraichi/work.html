第1章：SIer時代（2019年4月〜2021年8月）

業務内容

  • ECサイトの運用 - 既存システムの保守・改修がメイン
  • 使用技術 - Java、JavaScript、SQL
  • 開発スタイル - ウォーターフォール型の開発プロセス

SIerで学んだこと

  • ✓ 基礎的な開発フロー - 要件定義、設計、実装、テスト、リリースの一連の流れ
  • ✓ SQLの実践 - 大規模データベースを扱う実務経験
  • ✓ チーム開発 - 複数人でのコード管理とコミュニケーション
  • ✓ 品質管理 - テスト仕様書の作成とバグ修正

SIer時代の課題

  • レガシーなシステムが多く、新しい技術に触れる機会が少ない
  • ウォーターフォール型で開発サイクルが長い
  • モダンなWeb開発の経験を積みたいという思いが強くなった

第2章：外資系Web企業時代（2021年8月〜2024年3月）

転職のきっかけ

知人からの紹介で、外資系Web企業に転職のチャンスが訪れた。 一回の面接で採用が決まり、モダンな技術スタックを扱える環境に飛び込むことができた。

使用技術スタック

バックエンド
  • • Spring Boot（Java）
  • • Laravel（PHP）
フロントエンド
  • • React（JavaScript/TypeScript）
インフラ・クラウド
  • • AWS（Amazon Web Services）
  • • Azure（Microsoft Azure）
  • • GCP（Google Cloud Platform）
その他
  • • Docker / Kubernetes
  • • CI/CD（GitHub Actions等）

外資Web企業で成長したこと

  • モダンな技術スタック - React、Spring Boot、Laravelなど最新の技術に触れることができた
  • クラウド技術 - AWS、Azure、GCPの3大クラウドを実務で使用
  • アジャイル開発 - スピード感のある開発サイクルを経験
  • 国際的な環境 - 英語でのコミュニケーションやドキュメント作成

SIerとの最大の違い： 技術選定の自由度が高く、最新技術をどんどん取り入れる文化があった。 また、開発スピードが圧倒的に速く、週単位でリリースを繰り返す環境だった。

第3章：フリーランスエンジニアへ（2024年3月〜）

フリーランスになった理由

  • 自由な働き方 - 場所や時間に縛られない働き方を実現したい
  • 自分のサービスを作る時間 - 会社員時代は業務に追われて個人開発の時間が取れなかった
  • 収入の向上 - スキルを活かして収入を増やしたい
  • 多様な経験 - 様々なプロジェクトに関わることでスキルの幅を広げたい

フリーランスとして意識していること

  • ✓ 継続的な学習 - 常に最新技術をキャッチアップ
  • ✓ 情報発信 - Udemy講座、SNS、ブログなどで知見を共有
  • ✓ 人脈作り - エンジニアコミュニティへの参加
  • ✓ 健康管理 - 自己管理能力が重要

営業11年の経験はエンジニアに活きたか？

三菱自動車で11年間営業をしていた経験は、エンジニアとしても大いに役立っている。

営業経験が活きた場面

  • コミュニケーション能力 - クライアントや他部署との折衝がスムーズ
  • 提案力 - 技術的な解決策をビジネス視点で提案できる
  • 顧客視点 - ユーザーが本当に求めているものを理解する力
  • プレゼン能力 - 技術的な内容をわかりやすく説明できる
  • 交渉力 - 要件調整や納期交渉などで発揮

結論： 営業経験は決して無駄ではなかった。むしろ、技術力とビジネススキルの両方を持つことで、 他のエンジニアとの差別化につながっている。
