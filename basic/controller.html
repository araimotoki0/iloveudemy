コントローラ（Controller）とは

コントローラとは、アプリケーションのロジック（処理）をまとめて管理するクラスのことです。 これまではルートファイル（routes/web.php）に直接処理を書いていましたが、 アプリケーションが大きくなると管理が大変になります。

コントローラを使うことで、ルートファイルをシンプルに保ち、処理を整理して管理できます。

MVCアーキテクチャ

LaravelはMVC（Model-View-Controller）アーキテクチャを採用しています。

  • Model（モデル） - データベースとのやり取りを担当
  • View（ビュー） - 画面表示を担当
  • Controller（コントローラ） - ビジネスロジックを担当

役割を分けることで、コードの見通しが良くなり、保守性が向上します。

コントローラを使わない場合の問題点

まず、コントローラを使わない場合のコードを見てみましょう。

// routes/web.php

Route::get('/users', function () {
    $users = ['太郎', '花子', '次郎'];
    return view('users.index', compact('users'));
});

Route::get('/users/{id}', function ($id) {
    // ユーザー情報を取得する処理
    $user = ['id' => $id, 'name' => '太郎'];
    return view('users.show', compact('user'));
});

Route::post('/users', function () {
    // ユーザー作成処理
    // バリデーション処理
    // データベース保存処理
    return redirect('/users');
});

問題点：

  • routes/web.phpが肥大化する
  • 処理が分散して管理しづらい
  • 再利用しづらい
  • テストがしづらい

コントローラの作成

Artisanコマンドで作成

コントローラは、Artisanコマンドで簡単に作成できます。

php artisan make:controller UserController

これにより、app/Http/Controllers/UserController.php が作成されます。

作成されたファイルの中身

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    //
}

Controller クラスを継承した空のクラスが作成されます。
ここにメソッド（処理）を追加していきます。

基本的なコントローラの使い方

1. コントローラにメソッドを追加

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    // ユーザー一覧を表示
    public function index()
    {
        $users = ['太郎', '花子', '次郎'];
        return view('users.index', compact('users'));
    }

    // 個別ユーザーを表示
    public function show($id)
    {
        $user = ['id' => $id, 'name' => '太郎'];
        return view('users.show', compact('user'));
    }
}

2. ルートからコントローラを呼び出す

// routes/web.php

use App\Http\Controllers\UserController;

Route::get('/users', [UserController::class, 'index']);
Route::get('/users/{id}', [UserController::class, 'show']);

解説：

  • [コントローラクラス::class, 'メソッド名'] の形式で指定
  • ファイル先頭で use でコントローラをインポート
  • ルートファイルがすっきりする

✅ ルートファイルは「交通整理」だけを担当し、実際の処理はコントローラに任せます。

リソースコントローラ

CRUD操作（作成・読取・更新・削除）を行うコントローラは、 リソースコントローラとして一括で作成できます。

リソースコントローラを作成

php artisan make:controller UserController --resource

自動的に7つのメソッドが作成されます：

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    // 一覧表示
    public function index()
    {
        //
    }

    // 新規作成フォーム表示
    public function create()
    {
        //
    }

    // データ保存
    public function store(Request $request)
    {
        //
    }

    // 個別データ表示
    public function show(string $id)
    {
        //
    }

    // 編集フォーム表示
    public function edit(string $id)
    {
        //
    }

    // データ更新
    public function update(Request $request, string $id)
    {
        //
    }

    // データ削除
    public function destroy(string $id)
    {
        //
    }
}

リソースルートの定義

7つのルートを1行で定義できます。

// routes/web.php

use App\Http\Controllers\UserController;

Route::resource('users', UserController::class);

この1行で、以下のルートが自動的に定義されます：

HTTPメソッド URI アクション 用途
GET /users index 一覧表示
GET /users/create create 新規作成フォーム
POST /users store データ保存
GET /users/{id} show 個別表示
GET /users/{id}/edit edit 編集フォーム
PUT/PATCH /users/{id} update データ更新
DELETE /users/{id} destroy データ削除

💡 php artisan route:list で定義されたルートを確認できます。

Requestオブジェクトの利用

フォームから送信されたデータを受け取るには、Request オブジェクトを使います。 

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    public function store(Request $request)
    {
        // 全てのデータを取得
        $allData = $request->all();

        // 特定のデータを取得
        $name = $request->input('name');
        $email = $request->input('email');

        // デフォルト値を指定
        $age = $request->input('age', 20);

        // 一部のデータのみ取得
        $data = $request->only(['name', 'email']);

        // 特定のデータ以外を取得
        $data = $request->except(['password']);

        return '登録完了';
    }
}

バリデーション（入力チェック）

public function store(Request $request)
{
    $validated = $request->validate([
        'name' => 'required|max:255',
        'email' => 'required|email|unique:users',
        'age' => 'required|integer|min:0|max:120',
    ]);

    // バリデーションOKの場合のみここに到達
    // データベース保存処理など

    return redirect('/users');
}

主なバリデーションルール：

  • required - 必須
  • email - メール形式
  • max:n - 最大文字数
  • min:n - 最小値
  • integer - 整数
  • unique:テーブル名 - 一意性チェック

リダイレクトとレスポンス

リダイレクト

// URLへリダイレクト
return redirect('/users');

// ルート名でリダイレクト
return redirect()->route('users.index');

// 戻る
return back();

// フラッシュメッセージ付きリダイレクト
return redirect('/users')->with('success', '登録しました');

JSONレスポンス（API用）

// JSONを返す
return response()->json([
    'name' => '太郎',
    'email' => 'taro@example.com'
]);

// ステータスコード指定
return response()->json(['error' => 'Not Found'], 404);

ダウンロードレスポンス

// ファイルダウンロード
return response()->download($pathToFile);

// ファイル名を指定
return response()->download($pathToFile, 'filename.pdf');

依存性注入（Dependency Injection）

Laravelのコントローラでは、メソッドの引数に型を指定するだけで、 必要なオブジェクトが自動的に渡されます（依存性注入）。 

use Illuminate\Http\Request;

public function store(Request $request)
{
    // Requestオブジェクトが自動的に渡される
    $name = $request->input('name');
}

これはサービスコンテナという仕組みによって実現されています。
今は「型を書けば自動で渡される」と覚えておきましょう。

シングルアクションコントローラ

1つの処理だけを行うコントローラを作りたい場合は、 シングルアクションコントローラを使います。

作成方法

php artisan make:controller SendEmailController --invokable
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class SendEmailController extends Controller
{
    /**
     * Handle the incoming request.
     */
    public function __invoke(Request $request)
    {
        // メール送信処理
        return 'メールを送信しました';
    }
}

ルート定義

// routes/web.php

use App\Http\Controllers\SendEmailController;

Route::post('/send-email', SendEmailController::class);

メソッド名を指定せず、コントローラクラスだけを指定します。
自動的に __invoke() メソッドが呼ばれます。

コントローラの整理

サブディレクトリで整理

コントローラが増えてきたら、サブディレクトリで整理できます。

app/Http/Controllers/
├── Admin/
│   ├── UserController.php
│   └── PostController.php
├── Api/
│   └── UserController.php
└── UserController.php
# サブディレクトリ付きで作成
php artisan make:controller Admin/UserController
// routes/web.php

use App\Http\Controllers\Admin\UserController;

Route::get('/admin/users', [UserController::class, 'index']);

実践例：ユーザー管理機能

実際にユーザー管理機能を作る例を見てみましょう。

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    // 一覧表示
    public function index()
    {
        $users = [
            ['id' => 1, 'name' => '太郎', 'email' => 'taro@example.com'],
            ['id' => 2, 'name' => '花子', 'email' => 'hanako@example.com'],
        ];

        return view('users.index', compact('users'));
    }

    // 新規作成フォーム
    public function create()
    {
        return view('users.create');
    }

    // データ保存
    public function store(Request $request)
    {
        $validated = $request->validate([
            'name' => 'required|max:255',
            'email' => 'required|email',
        ]);

        // ここでデータベースに保存（次の講座で学びます）

        return redirect()->route('users.index')
            ->with('success', 'ユーザーを作成しました');
    }

    // 個別表示
    public function show($id)
    {
        $user = ['id' => $id, 'name' => '太郎', 'email' => 'taro@example.com'];

        return view('users.show', compact('user'));
    }

    // 編集フォーム
    public function edit($id)
    {
        $user = ['id' => $id, 'name' => '太郎', 'email' => 'taro@example.com'];

        return view('users.edit', compact('user'));
    }

    // データ更新
    public function update(Request $request, $id)
    {
        $validated = $request->validate([
            'name' => 'required|max:255',
            'email' => 'required|email',
        ]);

        // ここでデータベースを更新

        return redirect()->route('users.show', $id)
            ->with('success', 'ユーザーを更新しました');
    }

    // データ削除
    public function destroy($id)
    {
        // ここでデータベースから削除

        return redirect()->route('users.index')
            ->with('success', 'ユーザーを削除しました');
    }
}
// routes/web.php

use App\Http\Controllers\UserController;

Route::resource('users', UserController::class);

まとめ

この章では、Laravelのコントローラ（Controller）について学びました。

重要なポイント

  • コントローラは app/Http/Controllers/ に配置
  • php artisan make:controller で作成
  • ルートファイルから [クラス::class, 'メソッド'] で呼び出し
  • --resource オプションでCRUD操作の7メソッドを自動生成
  • Route::resource() で7つのルートを一括定義
  • Request オブジェクトでフォームデータを取得
  • validate() で入力チェック
