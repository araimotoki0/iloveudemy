🚢 コンテナって何？

前のセクションでdocker container run hello-worldを実行しました。 このとき、イメージからコンテナが作られて実行されました。

コンテナ = 「イメージを実行している状態」

🍱 お弁当に例えると...

前のセクションで「イメージ = 完成したお弁当」と説明しました。

  • • イメージ = 完成したお弁当（箱に入った状態）
  • • コンテナ = お弁当を開いて食べている状態

同じお弁当（イメージ）から、何人でも同時に食べる（コンテナを起動する）ことができます！

📝 イメージとコンテナの関係

  • • イメージ - 設計図（テンプレート）
  • • コンテナ - 設計図から作った実体

1つのイメージから複数のコンテナを作ることができます。

💡 コンテナの特徴

  • ✅ 独立した実行環境 - 他のコンテナやホストに影響しない
  • ✅ 起動が速い - 仮想マシンより圧倒的に速い（秒単位）
  • ✅ 使い捨て可能 - 削除しても元のイメージは残る

🎓 コンテナ操作コマンドを学ぼう

コンテナの概念が分かったところで、次は実際にコマンドを打ちながらコンテナを操作していきます。

❓ なぜコマンドを学ぶのか

最終的にはdocker composeで自動化しますが、その前に基礎コマンドを理解しておくと：

  • • docker-compose.yml の設定が何をしているか分かる
  • • トラブル時に原因を特定できる
  • • コンテナの状態を確認・操作できる

📋 これから学ぶコマンド

コマンド できること
docker container ls コンテナ一覧を見る
docker container run コンテナを作成・起動
docker container stop コンテナを停止
docker container start 停止したコンテナを再起動

では、Step 1から順番にやっていきましょう！

🖥️ Step 1: コンテナ一覧を見る

Step 1/6 docker container ls

まずは現在のコンテナの状態を確認しましょう。

1実行中のコンテナを表示

docker container ls
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES

何も表示されない？それは実行中のコンテナがないからです。

2全てのコンテナを表示（停止中も含む）

docker container ls -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES be1937c0a0c8 hello-world "/hello" 10 minutes ago Exited (0) 10 minutes ago eager_bell

✅ 前に実行したhello-worldコンテナが見つかりました！

📋 表示される項目の意味

  • CONTAINER ID: コンテナの一意のID
  • IMAGE: コンテナの元になったイメージ
  • COMMAND: 実行されたコマンド
  • CREATED: 作成された日時
  • STATUS: 現在の状態（Up=実行中、Exited=停止）
  • PORTS: ポートマッピング
  • NAMES: コンテナの名前（自動生成）

🎯 学んだコマンド

  • docker container ls - 実行中のコンテナを表示
  • docker container ls -a - 全てのコンテナを表示（停止中も含む）

🚀 Step 2: コンテナを起動する

Step 2/6 docker container run

今度はずっと動き続けるコンテナを起動してみましょう。 Webサーバーの「nginx」を使います。

1nginxコンテナをバックグラウンドで起動

docker container run -d -p 8080:80 --name my-nginx nginx
Unable to find image 'nginx:latest' locally latest: Pulling from library/nginx ... Status: Downloaded newer image for nginx:latest a1b2c3d4e5f6...

📋 オプションの意味

  • -d: デタッチモード（バックグラウンドで実行）
  • -p 8080:80: ホストの8080番ポート → コンテナの80番ポート
  • --name my-nginx: コンテナに名前を付ける

2ブラウザで確認

ブラウザで http://localhost:8080 を開いてみてください。

🎊 Webサーバーが動いた！

「Welcome to nginx!」が表示されたら成功です！

3実行中のコンテナを確認

docker container ls
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES a1b2c3d4e5f6 nginx "/docker-entrypoint.…" 30 seconds ago Up 29 seconds 0.0.0.0:8080->80/tcp my-nginx

✅ STATUSが「Up」になっている = 実行中です！

🎯 学んだコマンド

docker container run [オプション] イメージ名 - コンテナを作成・起動

⏹️ Step 3: コンテナを停止・再起動

Step 3/6 docker container stop / start

実行中のコンテナを停止してみましょう。

1コンテナを停止

docker container stop my-nginx
my-nginx

ブラウザで http://localhost:8080 にアクセスすると、もう表示されません。

2停止を確認

docker container ls -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES a1b2c3d4e5f6 nginx "/docker-entrypoint.…" 2 minutes ago Exited (0) 10 seconds ago my-nginx

STATUSが「Exited」= 停止しています

3コンテナを再起動

docker container start my-nginx
my-nginx

✅ 再び http://localhost:8080 にアクセスできるようになりました！

💡 stop と start の違い

  • stop: コンテナを停止（データは保持される）
  • start: 停止したコンテナを再起動

停止したコンテナは削除しない限り残っているので、startでいつでも再起動できます。

🎯 学んだコマンド

  • docker container stop コンテナ名 - コンテナを停止
  • docker container start コンテナ名 - コンテナを起動

📜 Step 4: コンテナのログを見る

Step 4/6 docker container logs

コンテナが何をしているか、ログを見てみましょう。

1ログを表示

docker container logs my-nginx
/docker-entrypoint.sh: /docker-entrypoint.d/ is not empty, will attempt to perform configuration ... 2024/01/15 10:30:00 [notice] 1#1: start worker processes 172.17.0.1 - - [15/Jan/2024:10:30:15 +0000] "GET / HTTP/1.1" 200 615 "-" "Mozilla/5.0..."

ブラウザからのアクセス履歴も確認できます。

2リアルタイムでログを追跡

docker container logs -f my-nginx

-fオプションでリアルタイム表示。ブラウザを更新すると新しいログが流れます。

終了するには Ctrl + C を押してください。

🎯 学んだコマンド

  • docker container logs コンテナ名 - ログを表示
  • docker container logs -f コンテナ名 - リアルタイムでログを追跡

🔧 Step 5: コンテナ内でコマンドを実行

Step 5/6 docker container exec

実行中のコンテナの中に入って、コマンドを実行できます。

1コンテナ内でコマンドを実行

docker container exec my-nginx cat /etc/nginx/nginx.conf

nginxの設定ファイルの中身が表示されます。

2コンテナ内でシェルを起動

docker container exec -it my-nginx /bin/bash
root@a1b2c3d4e5f6:/#

✅ コンテナの中に入りました！

📋 オプションの意味

  • -i: 標準入力を開いたままにする（interactive）
  • -t: 疑似ターミナルを割り当てる（tty）

シェルを起動するときは-itをセットで使います。

3コンテナ内で操作してみる

# コンテナ内で実行
ls /usr/share/nginx/html
50x.html index.html

nginxのドキュメントルートが見えます。

4コンテナから出る

exit

✅ コンテナから出ました。コンテナは引き続き実行中です。

🎯 学んだコマンド

  • docker container exec コンテナ名 コマンド - コンテナ内でコマンド実行
  • docker container exec -it コンテナ名 /bin/bash - コンテナ内でシェル起動

🗑️ Step 6: コンテナを削除

Step 6/6 docker container rm / prune

不要なコンテナを削除しましょう。

1コンテナを停止してから削除

# まず停止
docker container stop my-nginx
# 削除
docker container rm my-nginx
my-nginx

2削除を確認

docker container ls -a

✅ my-nginxが削除されました！

⚠️ 実行中のコンテナは削除できない

実行中のコンテナを削除しようとするとエラーになります。

# 強制削除する場合
docker container rm -f コンテナ名

3停止中のコンテナを一括削除

docker container prune
WARNING! This will remove all stopped containers. Are you sure you want to continue? [y/N] y Deleted Containers: be1937c0a0c8... Total reclaimed space: 0B

⚠️ hello-worldなど停止中のコンテナが全て削除されます

🎯 学んだコマンド

  • docker container rm コンテナ名 - コンテナを削除
  • docker container rm -f コンテナ名 - 強制削除
  • docker container prune - 停止中のコンテナを一括削除

⚡ 便利なオプション

docker container runでよく使うオプションを紹介します。

📋 よく使うオプション

オプション 説明 例
-d バックグラウンドで実行 -d
-p ポートマッピング -p 8080:80
--name コンテナに名前を付ける --name my-app
-e 環境変数を設定 -e MYSQL_ROOT_PASSWORD=secret
-v ボリュームをマウント -v ./data:/var/lib/mysql
--rm 停止時に自動削除 --rm
-it 対話モードで起動 -it ubuntu /bin/bash

🎯 オプションの組み合わせ例

# Webサーバーを起動
docker container run -d -p 8080:80 --name web nginx
# 対話モードでUbuntuを起動（終了時に自動削除）
docker container run -it --rm ubuntu /bin/bash
# MySQLを起動（パスワード設定）
docker container run -d --name db -e MYSQL_ROOT_PASSWORD=secret mysql
