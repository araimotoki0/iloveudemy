🎯 Dockerイメージとは

Dockerイメージは、コンテナを作成するための「設計図」または「テンプレート」です。

📋

イメージの役割

  • • OSやアプリケーションの実行環境
  • • 必要なファイルやライブラリを含む
  • • 読み取り専用（変更不可）
  • • 何度でもコンテナを作成できる
🏗️

レイヤー構造

イメージは複数のレイヤー（層）が積み重なって構成されています。

アプリケーション層
ライブラリ層
ミドルウェア層
OS基本層
🔍 レイヤー構造のメリット
効率的な保存

共通のレイヤーは再利用されるため、ディスク容量を節約できます

高速なダウンロード

既存のレイヤーは再ダウンロード不要です

キャッシュ活用

ビルド時に変更のないレイヤーはキャッシュを利用

🔍 イメージの検索と取得

Docker Hubとは

Docker Hubは、Dockerイメージを共有・配布するための公式レジストリ（リポジトリ）です。

公式イメージ

Docker社や各プロジェクトが管理する信頼性の高いイメージ

• nginx
• mysql
• python
• node
コミュニティイメージ

個人やコミュニティが作成したイメージ

• username/myapp
• organization/tool

docker pull - イメージの取得

Docker Hubからイメージをローカルにダウンロードします。

基本形
docker pull nginx
💡 最新版（latest）を取得
docker pull nginx:1.24
💡 バージョン1.24を取得
タグの指定

タグを使って特定のバージョンや環境を指定できます

nginx:latest 最新版
nginx:1.24 バージョン指定
nginx:alpine Alpine Linux版（軽量）
python:3.11-slim Python 3.11のスリム版
⚠️ latestタグの注意点

「latest」は必ずしも最新版とは限りません。本番環境では必ず具体的なバージョンタグを指定することを推奨します。

🗂️ イメージの管理

docker images - イメージ一覧の表示

ローカルに保存されているイメージを確認します。

docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
nginx latest a6bd71f48f68 2 weeks ago 187MB
nginx 1.24 f8f4ffc8092c 3 weeks ago 186MB
python 3.11 a5d7930b60cc 1 month ago 1.01GB
python 3.11-slim ee7d5c57a2cd 1 month ago 126MB
ubuntu 22.04 ca2b0f26964c 2 months ago 77.9MB
🔍 各カラムの意味
REPOSITORY: イメージの名前
TAG: バージョンやバリエーション
IMAGE ID: イメージの一意な識別子（短縮形）
CREATED: イメージが作成された日時
SIZE: イメージのサイズ（圧縮前）

docker image inspect - 詳細情報の確認

イメージの詳細な情報をJSON形式で表示します。

docker image inspect nginx
💡 設定、レイヤー、環境変数などを確認
よく確認する情報
  • • Layers: レイヤー構成
  • • Env: 環境変数
  • • ExposedPorts: 公開ポート
  • • Cmd: デフォルトコマンド
  • • WorkingDir: 作業ディレクトリ

🗑️ イメージの削除とクリーンアップ

docker rmi - イメージの削除

不要なイメージを削除してディスク容量を節約します。

docker rmi nginx
💡 イメージ名で削除
docker rmi nginx:1.24
💡 特定のタグを指定して削除
docker rmi a6bd71f48f68
💡 IMAGE IDで削除
docker rmi -f nginx
💡 強制削除（コンテナが使用中でも削除）
⚠️ 削除できない場合

以下の場合、イメージは削除できません：

  • • コンテナが使用中（実行中または停止中）
  • • 別のイメージが依存している

解決方法: 先にコンテナを削除してからイメージを削除します。

ダングリングイメージとは

タグがなく、どのコンテナからも参照されていないイメージのことです。

docker images -f dangling=true
💡 ダングリングイメージを表示
REPOSITORY TAG IMAGE ID CREATED SIZE
<none> <none> 3a5e8d7c9f1a 3 days ago 250MB
<none> <none> b8c4d5e2f6a9 1 week ago 180MB
docker image prune
💡 ダングリングイメージを一括削除

未使用イメージの一括削除

docker image prune -a
💡 使用されていないイメージをすべて削除
docker system prune -a --volumes
💡 未使用のイメージ、コンテナ、ボリューム、ネットワークを全て削除
🚨 注意

docker image prune -a や docker system prune -a は、使用していないすべてのイメージを削除します。実行前に必ず内容を確認してください。

💾 イメージの保存と共有

docker save - イメージをファイルに保存

イメージをtarファイルとしてエクスポートします。オフライン環境への移行などに便利です。

docker save nginx -o nginx.tar
💡 nginxイメージをnginx.tarに保存
docker save nginx:1.24 nginx:latest -o nginx-images.tar
💡 複数のタグをまとめて保存

docker load - ファイルからイメージを読み込み

tarファイルからイメージをインポートします。

docker load -i nginx.tar
💡 nginx.tarからイメージを読み込み
docker load < nginx.tar
💡 標準入力からの読み込み
💡 使用例
オフライン環境への移行

インターネット接続のない環境でもイメージを使用できます

バックアップ

重要なイメージを保存しておけます

配布

チーム内でイメージを共有できます

docker tag - イメージにタグを付ける

既存のイメージに新しい名前やタグを付けます。

docker tag nginx myregistry/nginx:v1.0
💡 プライベートレジストリ用にタグ付け
docker tag nginx:latest nginx:backup
💡 バックアップ用のタグを作成

docker push - イメージをレジストリに公開

イメージをDocker Hubやプライベートレジストリにアップロードします。

事前準備
docker login
💡 Docker Hubにログイン
docker push username/myapp:v1.0
💡 イメージをDocker Hubにプッシュ
📝 注意

pushするには、イメージ名が「username/repository:tag」の形式である必要があります。usernameはDocker Hubのアカウント名です。
