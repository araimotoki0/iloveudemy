📦 使用するリポジトリ

https://github.com/sbtest1218-oss/udemy-aws3-laravel-new

📂 プロジェクト構造を理解する

このプロジェクトは、本番環境も想定したプロフェッショナルな構成になっています。

プロジェクトルート/ ├── infra/ # Docker関連ファイル │ ├── php/ │ │ ├── Dockerfile # PHPカスタムイメージ │ │ └── php.ini # PHP設定 │ ├── mysql/ │ │ ├── Dockerfile # MySQLカスタムイメージ │ │ └── my.cnf # MySQL設定 │ └── nginx/ │ └── default.conf # Nginx設定 ├── docker-compose.yml # 開発環境用 ├── docker-compose.prod.yml # 本番環境用 ├── .env.example # 環境変数テンプレート └── src/ # Laravelプロジェクト

🏗️ 構成の特徴

  • infraディレクトリ: Docker関連ファイルを分離（見通しが良い）
  • カスタムDockerfile: 必要なPHP拡張やツールを事前インストール
  • 環境別設定: 開発用と本番用でdocker-compose.ymlを分離
  • 設定ファイル分離: php.ini、my.cnf、default.confで細かく調整

Step 1: GitHubからプロジェクトを取得

リポジトリをクローン

# プロジェクトをクローン git clone https://github.com/sbtest1218-oss/udemy-aws3-laravel-new.git cd udemy-aws3-laravel-new

💡 自分のリポジトリにする場合

GitHubで「Use this template」ボタンを押して、自分のアカウントにコピーしてから使うこともできます。その場合は自分のリポジトリURLでcloneしてください。

環境変数ファイルを作成

# .env.exampleをコピー cp .env.example .env

.envファイルには、データベース接続情報などが含まれています。必要に応じて編集してください。

Step 2: docker-compose.ymlを理解する

プロジェクトのdocker-compose.ymlを見てみましょう。

appサービス（PHP-FPM）

app: build: context: ./infra/php user: "1000:1000" working_dir: /data volumes: - ./src:/data ports: - "5173:5173" # Vite開発サーバー用 depends_on: - db environment: DB_HOST: db DB_PORT: 3306 DB_DATABASE: laravel_db DB_USERNAME: laravel_user DB_PASSWORD: laravel_pass restart: unless-stopped

🔍 重要なポイント

  • build: ./infra/php: カスタムDockerfileでイメージをビルド
  • user: "1000:1000": ファイル権限の問題を回避（ホストユーザーと一致）
  • ports: 5173: Vite（フロントエンドビルドツール）用のポート
  • restart: unless-stopped: コンテナが落ちたら自動再起動

webサービス（Nginx）

web: image: nginx:1.25-alpine ports: - "80:80" volumes: - ./src:/data - ./infra/nginx/default.conf:/etc/nginx/conf.d/default.conf depends_on: - app working_dir: /data restart: unless-stopped

💡 Nginxは公式イメージを使用

Nginxは設定ファイル（default.conf）をマウントするだけで十分なので、カスタムDockerfileは不要です。

dbサービス（MySQL）

db: build: context: ./infra/mysql ports: - "3306:3306" volumes: - db-store:/var/lib/mysql environment: MYSQL_DATABASE: laravel_db MYSQL_USER: laravel_user MYSQL_PASSWORD: laravel_pass MYSQL_ROOT_PASSWORD: root TZ: 'Asia/Tokyo' restart: unless-stopped
  • ports: 3306: ホストから直接MySQLにアクセス可能（開発時に便利）
  • TZ: 'Asia/Tokyo': タイムゾーンを日本時間に設定
  • db-store: 名前付きボリュームでデータ永続化

ネットワーク

networks: app-network: driver: bridge

すべてのサービスがapp-networkで接続され、サービス名で通信できます。

Step 3: PHPのDockerfileを理解する

infra/php/Dockerfileを見てみましょう。

FROM php:8.3-fpm-bookworm # Composerをインストール COPY --from=composer:2.7 /usr/bin/composer /usr/bin/composer # 必要なパッケージをインストール RUN apt-get update && apt-get install -y \ git \ unzip \ libzip-dev \ libicu-dev \ libonig-dev \ # 画像処理用 libfreetype6-dev \ libjpeg62-turbo-dev \ libpng-dev # Node.js 24.xをインストール（フロントエンド用） RUN curl -fsSL https://deb.nodesource.com/setup_24.x | bash - \ && apt-get install -y nodejs # PHP拡張をインストール RUN docker-php-ext-install \ intl \ # 国際化 pdo_mysql \ # MySQLデータベース接続 zip \ # ZIP圧縮 bcmath # 高精度計算 # GD拡張（画像処理）を設定してインストール RUN docker-php-ext-configure gd --with-freetype --with-jpeg \ && docker-php-ext-install gd # php.iniをコピー COPY php.ini /usr/local/etc/php/php.ini # 専用ユーザーを作成（権限問題の回避） RUN useradd -m -u 1000 -s /bin/bash appuser WORKDIR /data

✨ このDockerfileのすごいところ

  • Composer内蔵: 別途インストール不要
  • Node.js内蔵: フロントエンドのビルドもできる
  • 必要なPHP拡張がすべて揃っている: Laravel開発に必要な機能完備
  • 権限問題を回避: ホストユーザーと同じUID/GIDのユーザーを作成

🔍 各PHP拡張の役割

  • intl: 多言語対応（日本語処理など）
  • pdo_mysql: MySQLデータベースへの接続（必須）
  • zip: ZIPファイルの作成・展開（Composer依存）
  • bcmath: 高精度な数値計算（金額計算など）
  • gd: 画像の加工・サムネイル生成

Step 4: コンテナを起動してLaravelをインストール

イメージをビルドしてコンテナ起動

# イメージをビルドして起動 docker compose up -d --build

初回は時間がかかります（5〜10分程度）。以下の処理が実行されます：

  • PHPイメージのビルド（Composer、Node.js、PHP拡張のインストール）
  • MySQLイメージのビルド
  • Nginxイメージのダウンロード
  • 3つのコンテナの起動

appコンテナに入る

docker compose exec app bash

Laravelをインストール

# Laravelをカレントディレクトリ（/data = src/）にインストール composer create-project laravel/laravel:"^12.0" . # 権限を設定 chmod -R 777 storage bootstrap/cache

⚠️ 最後のドット（.）を忘れずに

.は「現在のディレクトリにインストール」を意味します。これがないと、laravelという名前のサブディレクトリが作られてしまいます。

一旦コンテナから出る

exit

Step 5: Laravelのデータベース設定

src/.envファイルを開き、データベース接続情報を設定します。

DB_CONNECTION=mysql DB_HOST=db DB_PORT=3306 DB_DATABASE=laravel_db DB_USERNAME=laravel_user DB_PASSWORD=laravel_pass

💡 これらの値は...

docker-compose.ymlのdbサービスの環境変数と一致させてください。

コンテナを再起動

# 設定を反映させるため、データベースも含めて再起動 docker compose down -v docker compose up -d

🌍 本番環境用の設定（docker-compose.prod.yml）

このプロジェクトには、本番環境用のdocker-compose.prod.ymlも含まれています。

主な違い

  • dbサービスがない: 外部のデータベース（AWS RDSなど）を使用
  • 環境変数を外部から取得: .envファイルからDB接続情報を読み込む
  • ポート公開が異なる: 必要最小限のポートのみ

本番用で起動する場合

# 開発用と本番用の両方を使って起動 docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d

⚠️ 本番環境での注意点

  • デバッグモードをOFFに（APP_DEBUG=false）
  • 強力なAPP_KEYを生成
  • データベースパスワードを複雑に
  • HTTPSを使用

📝 よく使うDockerコマンド（このプロジェクト用）

# コンテナを起動 docker compose up -d # コンテナを停止 docker compose down # コンテナを停止してボリュームも削除（データリセット） docker compose down -v # appコンテナに入る docker compose exec app bash # ログを確認 docker compose logs docker compose logs -f web # コンテナの状態確認 docker compose ps # イメージを再ビルドして起動 docker compose up -d --build

Laravel関連のコマンド（appコンテナ内で）

# マイグレーション php artisan migrate # マイグレーションをロールバック php artisan migrate:rollback # マイグレーションをリフレッシュ（全削除→再実行） php artisan migrate:fresh # キャッシュクリア php artisan cache:clear php artisan config:clear php artisan route:clear # Composerでパッケージインストール composer install # フロントエンドのビルド npm install npm run build # 本番用 npm run dev # 開発用（ホットリロード）
