🎯 Hello Docker!

環境構築お疲れさまでした！それでは実際にDockerを動かしてみましょう。 最初は定番の「Hello World」から始めます。

🎉 このセクションで学ぶこと

  • Dockerの基本的なワークフロー
  • イメージの取得（pull）
  • コンテナの実行（run）
  • 実行中のコンテナの確認
  • 停止したコンテナの確認と削除

🐳 最初のDockerコマンド

ターミナル（Windows: コマンドプロンプトまたはPowerShell、Mac/Linux: ターミナル）を開いて、 以下のコマンドを入力してください：

docker run hello-world
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
719385e32844: Pull complete
Digest: sha256:4f53e2564790c8e7856ec08e384732aa38dc43c52f02952483e3f003afbf23db
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

✅ 成功！

「Hello from Docker!」というメッセージが表示されましたか？

おめでとうございます！あなたは今、初めてのDockerコンテナを動かしました！

🔍 何が起こったの？

このシンプルなコマンド1行で、Dockerは裏で複雑な処理を自動的に行ってくれました。 順番に見ていきましょう：

docker run hello-world の実行フロー
1 ローカルでイメージを探す

Docker は最初に、あなたのPC内に「hello-world」イメージがあるか確認します。 初回なので見つかりません → "Unable to find image 'hello-world:latest' locally"

↓
2 Docker Hubからダウンロード

見つからなかったので、自動的にDocker Hub（インターネット上のイメージ倉庫）から 「hello-world」イメージをダウンロードします → "Pulling from library/hello-world"

↓
3 コンテナを作成

ダウンロードしたイメージから、新しいコンテナを作成します。 イメージ（設計図）→ コンテナ（実体）という流れです。

↓
4 コンテナを起動・実行

コンテナの中で指定されたプログラム（この場合はメッセージを表示するだけの小さなプログラム）を実行します。

↓
5 結果を表示して終了

"Hello from Docker!" というメッセージが画面に表示されます。 プログラムが終了すると、コンテナも自動的に停止します。

💡 重要なポイント

  • イメージは設計図・レシピのようなもの（1回ダウンロードすれば再利用可能）
  • コンテナはイメージから作られた実行環境（何度でも作成できる）
  • docker run は「イメージ取得 → コンテナ作成 → 起動」を一度に行う便利コマンド
  • プログラムが終わるとコンテナも自動的に停止する

🚀 もっと実用的なコンテナを動かそう

Hello Worldは理解できましたね。次はもっと実用的なコンテナを動かしてみましょう。 Webサーバー（nginx）を起動してみます。

🌐 nginxコンテナを起動

nginx（エンジンエックス）は、世界中で使われている高速なWebサーバーソフトウェアです。 通常なら複雑なインストール作業が必要ですが、Dockerなら一瞬です！

docker run -d -p 8080:80 --name my-nginx nginx

⚠️ オプションの意味

少し複雑に見えますが、1つずつ理解すれば簡単です：

  • -d - デタッチモード（バックグラウンドで実行）
  • -p 8080:80 - ポートフォワーディング（あなたのPC:8080 → コンテナ:80）
  • --name my-nginx - コンテナに名前を付ける
  • nginx - 使用するイメージ名

🔍 オプションを詳しく解説

📌 -d (detach mode)

バックグラウンドで実行する指定です。

  • -d を付けない → ターミナルがコンテナに占有される（終了するまで他のコマンドが打てない）
  • -d を付ける → コンテナは裏で動き続け、ターミナルは自由に使える

📌 -p 8080:80 (port forwarding)

ポート番号のマッピングを設定します。

あなたのPC:8080 → コンテナ:80

つまり：

  • ブラウザで localhost:8080 にアクセスすると...
  • その通信がコンテナの80番ポート（nginxが待ち受けているポート）に転送される
  • コンテナは隔離されているので、このマッピングがないと外からアクセスできない

📌 --name my-nginx

コンテナにわかりやすい名前を付けます。

  • 指定しないと、Dockerがランダムな名前を自動生成する（例: "silly_einstein"）
  • 名前を付けておくと、後で操作するときに便利

✅ 動作確認

コンテナが起動したら、ブラウザを開いて以下のURLにアクセスしてみてください：

http://localhost:8080

🎉 nginxのウェルカムページが表示されましたか？

"Welcome to nginx!" というページが表示されれば成功です！

これで、あなたのPC上でWebサーバーが動いています。 インストール不要、設定不要、たった1行のコマンドでWebサーバーが起動しました！

💡 すごいポイント

  • nginx本体のインストール不要（コンテナの中に全て入っている）
  • PCを汚さない（コンテナを削除すれば痕跡ゼロ）
  • 同じコマンドで誰でも同じ環境が作れる
  • バージョンを簡単に切り替えられる（nginx:1.21、nginx:1.22など）

📊 コンテナの状態を確認しよう

今、あなたのPC上でnginxコンテナが動いています。 Dockerコンテナの状態を確認する方法を学びましょう。

🔍 実行中のコンテナを確認

現在動いているコンテナの一覧を表示します：

docker ps
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "/docker-entrypoint.…" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx

📋 表示項目の意味

  • CONTAINER ID - コンテナの一意な識別子（短縮版）
  • IMAGE - 使用しているイメージ名
  • COMMAND - コンテナ内で実行されているコマンド
  • CREATED - コンテナが作成された時刻
  • STATUS - 現在の状態（Up = 実行中）
  • PORTS - ポートのマッピング情報
  • NAMES - コンテナの名前（--nameで指定したもの）

📜 全てのコンテナを確認（停止中も含む）

停止しているコンテナも含めて、全てのコンテナを表示します：

docker ps -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "/docker-entrypoint.…" 5 minutes ago Up 5 minutes 0.0.0.0:8080->80/tcp my-nginx
xyz789ghi012 hello-world "/hello" 10 minutes ago Exited (0) 10 minutes ago romantic_tesla

⚠️ -a オプション

-a (all) を付けると、停止中のコンテナも表示されます。

hello-worldコンテナは「Exited (0)」つまり正常終了して停止している状態です。

🎯 特定のコンテナの詳細情報

コンテナの詳細情報をJSON形式で表示します：

docker inspect my-nginx

※ たくさんの情報が表示されますが、今は全部理解する必要はありません。 必要に応じて確認する程度でOKです。

🎮 コンテナを操作しよう

コンテナの基本的な操作（停止・起動・再起動・削除）を学びます。

⏸️ コンテナを停止

動いているコンテナを停止します：

docker stop my-nginx
my-nginx

停止したことを確認してみましょう：

docker ps
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES

→ 何も表示されません（実行中のコンテナがないため）

docker ps -a
CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "/docker-entrypoint.…" 10 minutes ago Exited (0) 5 seconds ago my-nginx

→ STATUSが「Exited」になっています

▶️ コンテナを起動

停止しているコンテナを再度起動します：

docker start my-nginx
my-nginx

もう一度ブラウザで http://localhost:8080 にアクセスしてみてください。 またnginxのページが表示されるはずです！

💡 docker run と docker start の違い

  • docker run - 新しいコンテナを作成して起動（イメージから）
  • docker start - 既存のコンテナを起動（停止中のものを再開）

🔄 コンテナを再起動

停止→起動を一度に行います：

docker restart my-nginx
my-nginx

🗑️ コンテナを削除

不要になったコンテナを削除します。まず停止してから削除します：

docker stop my-nginx
docker rm my-nginx
my-nginx

⚠️ 実行中のコンテナは削除できない

実行中のコンテナを削除しようとするとエラーになります：

docker rm my-nginx
Error response from daemon: You cannot remove a running container...

強制削除する場合は -f オプションを使います：

docker rm -f my-nginx

🧹 停止中のコンテナを一括削除

停止しているコンテナが溜まってきたら、一括削除できます：

docker container prune
WARNING! This will remove all stopped containers.
Are you sure you want to continue? [y/N] y
Deleted Containers:
xyz789ghi012
Total reclaimed space: 0B

🖼️ イメージを管理しよう

コンテナの元になる「イメージ」の管理方法を学びます。

📋 イメージ一覧を確認

あなたのPC上にあるDockerイメージを表示します：

docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
nginx latest abc123def456 2 weeks ago 142MB
hello-world latest xyz789ghi012 6 months ago 13.3kB

📋 表示項目の意味

  • REPOSITORY - イメージ名
  • TAG - バージョンタグ（latest = 最新版）
  • IMAGE ID - イメージの一意な識別子
  • CREATED - イメージが作成された日時
  • SIZE - イメージのサイズ

📥 イメージをダウンロード

Docker Hubから明示的にイメージをダウンロードします：

docker pull python:3.11
3.11: Pulling from library/python
abc123: Pull complete
def456: Pull complete
ghi789: Pull complete
Digest: sha256:abc123...
Status: Downloaded newer image for python:3.11

ダウンロードされたか確認：

docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
python 3.11 jkl012mno345 1 week ago 917MB
nginx latest abc123def456 2 weeks ago 142MB
hello-world latest xyz789ghi012 6 months ago 13.3kB

💡 docker pull と docker run の関係

docker run は自動的に docker pull を実行します。

  • 事前にpullしておく必要はない（runが自動でやってくれる）
  • ただし、大きいイメージは先にpullしておくと便利
  • 最新版に更新したい時は明示的にpullする

🗑️ イメージを削除

不要なイメージを削除してディスク容量を節約します：

docker rmi python:3.11
Untagged: python:3.11
Deleted: sha256:jkl012...
Deleted: sha256:mno345...

⚠️ 使用中のイメージは削除できない

そのイメージから作られたコンテナ（停止中も含む）が存在する場合、イメージは削除できません。

先にコンテナを削除してから、イメージを削除してください。

🧹 未使用イメージを一括削除

どのコンテナからも使われていないイメージを一括削除：

docker image prune

全ての未使用イメージを削除（使用中のもの以外全て）：

docker image prune -a
