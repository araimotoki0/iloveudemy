📦 Docker Desktopとは

Dockerを使うには、Docker Desktopというソフトウェアをインストールします。

💡 Docker Desktopって何？

Docker DesktopはDockerを簡単に使えるようにしてくれる統合パッケージです。以下が全て含まれています：

  • Docker Engine - Dockerの本体（コンテナを動かすエンジン）
  • Docker CLI - コマンドラインツール（ターミナルからDockerを操作）
  • Docker Compose - 複数コンテナを管理するツール
  • GUI（管理画面） - 視覚的にコンテナを管理できる
  • Kubernetes - オーケストレーションツール（オプション）

⚠️ 商用利用について

Docker Desktopは個人利用・小規模チーム（従業員250人未満かつ年間売上1000万ドル未満）は無料です。 大企業での使用には有料のサブスクリプションが必要になります。

※ 学習目的や個人開発では全く問題ありません！

📥 インストール手順

それでは実際にDocker Desktopをインストールしていきましょう！ あなたのOSに合わせたタブを選んでください。

🪟 Windows
🍎 macOS
🐧 Linux

1 WSL 2のインストール

WindowsでDockerを動かすには、まずWSL 2（Windows Subsystem for Linux 2）が必要です。

💡 WSL 2とは？

Windows上でLinuxを動かすための仕組みです。DockerはもともとLinux向けの技術なので、WindowsでもLinux環境が必要なんです。

WSL 2では軽量な仮想化技術を使って、Linuxカーネルを直接実行します。これによりDockerを高速に動かせます！

🔧 WSL 2のインストール手順

① PowerShellを管理者権限で開く

  • スタートメニューで「PowerShell」を検索
  • 右クリック → 「管理者として実行」を選択

② 以下のコマンドを実行

wsl --install

このコマンドは自動的に以下を行います：

  • WSL機能の有効化
  • 仮想マシンプラットフォームの有効化
  • Linuxカーネルコンポーネントのダウンロード
  • WSL 2をデフォルトに設定
  • Ubuntu（デフォルトのLinuxディストリビューション）のインストール

③ PCを再起動

インストールが完了したら、PCを再起動してください。

④ WSL 2の確認

再起動後、もう一度PowerShellを開いて以下を実行：

wsl --status
既定のバージョン: 2
既定のディストリビューション: Ubuntu

✓ 「既定のバージョン: 2」と表示されればOKです！

⚠️ エラーが出た場合

「仮想化が有効になっていません」というエラーが出たら：

  1. PCを再起動してBIOS/UEFI設定画面に入る（起動時にF2、Del、F12などを押す）
  2. 「Virtualization Technology」や「VT-x」「AMD-V」を探して有効化
  3. 設定を保存して再起動

2 Docker Desktopのダウンロード

公式サイトからDocker Desktopをダウンロードします。

ダウンロードURL:
https://www.docker.com/products/docker-desktop
  • 「Download for Windows」ボタンをクリック
  • インストーラー（Docker Desktop Installer.exe）がダウンロードされる
  • ファイルサイズは約500MB（インターネット回線によっては時間がかかります）

3 インストールの実行

ダウンロードしたインストーラーを実行します。

  • 「Docker Desktop Installer.exe」をダブルクリック
  • 「Use WSL 2 instead of Hyper-V」にチェックが入っていることを確認（推奨）
  • 「Add shortcut to desktop」もチェック推奨
  • 「OK」をクリックしてインストール開始
  • インストールには5-10分ほどかかります
📸 スクリーンショット：インストール設定画面
※ WSL 2オプションにチェックが入っていることを確認

4 Docker Desktopの起動と設定

インストールが完了したら、Docker Desktopを起動します。

  • インストール完了後、「Close and restart」をクリックしてPCを再起動
  • デスクトップの「Docker Desktop」アイコンをダブルクリック
  • 初回起動時はライセンス同意画面が表示される → 「Accept」
  • アンケート画面はスキップ可能
  • Docker Engineが起動するまで1-2分待つ

✓ 起動成功のサイン

  • タスクバーにDockerのクジラアイコンが表示される
  • アイコンが緑色になれば起動完了！

1 Macのチップ種類を確認

まず、あなたのMacがAppleシリコン（M1/M2/M3）かIntelかを確認します。

🔍 確認方法

  • 画面左上のAppleロゴをクリック
  • 「このMacについて」を選択
  • 「チップ」または「プロセッサ」の項目を確認

Appleシリコンの表示例

チップ: Apple M1
チップ: Apple M2 Pro
チップ: Apple M3 Max

Intelの表示例

プロセッサ: 2.3 GHz Intel Core i5
プロセッサ: 3.1 GHz Intel Core i7

2 Docker Desktopのダウンロード

チップに合ったバージョンをダウンロードします。

公式ダウンロードページ:
https://www.docker.com/products/docker-desktop

🍎 Appleシリコンの場合

「Download for Mac - Apple Chip」を選択

Docker Desktop for Mac (Apple Silicon)

💻 Intel Macの場合

「Download for Mac - Intel Chip」を選択

Docker Desktop for Mac (Intel)

⚠️ 重要

チップに合わない方をダウンロードすると動かないので注意してください！ ファイルサイズは約600MBです。

3 インストールの実行

ダウンロードしたDMGファイルからインストールします。

  • ダウンロードした「Docker.dmg」をダブルクリック
  • Dockerアイコンを「Applications」フォルダにドラッグ&ドロップ
  • コピーが完了するまで待つ（1-2分）
  • DMGウィンドウを閉じてOK
📸 スクリーンショット：DMGインストール画面
※ Dockerアイコン → Applicationsフォルダへドラッグ

4 Docker Desktopの起動

アプリケーションフォルダからDocker Desktopを起動します。

  • Finderで「アプリケーション」フォルダを開く
  • 「Docker」を探してダブルクリック
  • 「開いてもよろしいですか？」→「開く」をクリック
  • 初回起動時は権限の許可を求められる → パスワードを入力
  • ライセンス同意画面 → 「Accept」
  • Docker Engineの起動を待つ（1-2分）

✓ 起動成功のサイン

  • メニューバー（画面上部）にDockerのクジラアイコンが表示される
  • アイコンが動かなくなれば起動完了！

💡 Appleシリコン版の追加設定（推奨）

M1/M2/M3 Macを使っている場合、Rosetta 2エミュレーションを有効にすると、Intel用のイメージも動かせるようになります。

  • Docker Desktopの設定（⚙️アイコン）を開く
  • 「Features in development」→「Use Rosetta for x86/amd64 emulation」をオン
  • 「Apply & restart」で再起動

💡 LinuxでのDocker利用について

Linuxでは2つの選択肢があります：

  1. Docker Desktop for Linux - GUIツール付き（このコースで推奨）
  2. Docker Engine - CLI のみ（軽量だがGUIなし）

ここではDocker Desktop for Linuxのインストール方法を説明します。

1 Ubuntu/Debian でのインストール

① 古いバージョンの削除（既にインストールしている場合）

sudo apt-get remove docker docker-engine docker.io containerd runc

② 必要なパッケージのインストール

sudo apt-get update
sudo apt-get install ca-certificates curl gnupg

③ GPGキーの追加

sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

④ リポジトリの追加

echo \
"deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
$(. /etc/os-release && echo "$VERSION_CODENAME") stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

⑤ Docker Desktopのインストール

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

⑥ Docker Desktopのダウンロードとインストール

wget https://desktop.docker.com/linux/main/amd64/docker-desktop-4.26.1-amd64.deb
sudo apt-get install ./docker-desktop-4.26.1-amd64.deb

※ バージョン番号は最新のものに変わっている可能性があります。

公式サイト（https://docs.docker.com/desktop/install/linux-install/）で最新バージョンを確認してください。

2 ユーザー権限の設定

Dockerを一般ユーザーで使えるようにするため、dockerグループに自分を追加します。

sudo groupadd docker
sudo usermod -aG docker $USER

設定を反映するため、一度ログアウトして再ログイン

⚠️ セキュリティ注意

dockerグループに追加されたユーザーは、root権限と同等の権限を持ちます。 本番サーバーでは慎重に設定してください。

3 Docker Desktopの起動

アプリケーションメニューからDocker Desktopを起動します。

  • アプリケーションメニューで「Docker Desktop」を検索
  • クリックして起動
  • 初回起動時はライセンス同意画面 → 「Accept」
  • Docker Engineの起動を待つ

または、ターミナルから起動：

systemctl --user start docker-desktop

✓ システムトレイにDockerアイコンが表示されれば起動完了！

💡 docker-compose vs docker compose

古いバージョンでは「docker-compose」（ハイフン付き）というコマンドでしたが、 現在は「docker compose」（スペース）が推奨されています。

このコースでは新しい方（docker compose）を使います。

「Hello from Docker!」というメッセージが表示されましたか？

💡 何が起こったの？

このコマンドで以下のことが自動で行われました：

  1. イメージの検索 - ローカルにhello-worldイメージがないことを確認
  2. イメージのダウンロード - Docker Hubから自動的にダウンロード
  3. コンテナの作成 - イメージからコンテナを作成
  4. コンテナの実行 - メッセージを表示するプログラムを実行
  5. コンテナの終了 - 処理が終わったら自動的に停止

この一連の流れがDockerの基本動作です。次のセクションで詳しく学んでいきます！

🔧 トラブルシューティング

よくあるエラーと解決方法をまとめました。

❌ エラー: "docker: command not found"

bash: docker: command not found

🔍 原因

  • Dockerがインストールされていない
  • Docker Desktopが起動していない
  • PATHが通っていない

✅ 解決方法

  1. Docker Desktopが起動しているか確認
    • Windows: タスクバーにクジラアイコンがあるか
    • Mac: メニューバーにクジラアイコンがあるか
    • 起動していなければダブルクリックで起動
  2. ターミナル/コマンドプロンプトを再起動

    一度閉じて、もう一度開いてから試す

  3. PCを再起動

    それでもダメなら一度PCを再起動してみる

❌ エラー: "Cannot connect to the Docker daemon"

Cannot connect to the Docker daemon at unix:///var/run/docker.sock.
Is the docker daemon running?

🔍 原因

Docker Engineが起動していません。

✅ 解決方法

  1. Docker Desktopを起動

    アプリケーションからDocker Desktopを起動してください

  2. 起動完了まで待つ

    クジラアイコンが動かなくなるまで1-2分待ちます

  3. Windowsの場合：WSL 2の確認
    wsl --status

    WSL 2が起動しているか確認

❌ エラー: "Hardware assisted virtualization is not enabled"

Hardware assisted virtualization and data execution protection
must be enabled in the BIOS.

🔍 原因

BIOSで仮想化技術が無効になっています。

✅ 解決方法

  1. PCを再起動してBIOS/UEFI設定に入る
    • 起動時にF2、Del、F12、ESCキーなどを連打（PCメーカーによる）
    • ASUSならF2、DellならF2、HPならESCが多い
  2. 仮想化設定を探す

    BIOSの中で以下のような名前の項目を探します：

    • Intel CPU: "Intel Virtualization Technology" (VT-x)
    • AMD CPU: "AMD-V" または "SVM Mode"
    • 「Advanced」「CPU Configuration」などのメニューにあることが多い
  3. 有効化（Enabled）に変更

    DisabledまたはOffになっていればEnabledまたはOnに変更

  4. 設定を保存して終了

    F10キーで保存＆終了（Save and Exit）することが多い

⚠️ 注意

BIOS設定は慎重に行ってください。不明な項目は変更しないようにしましょう。 分からない場合は、お使いのPCのメーカー名＋機種名＋「仮想化 有効化」で検索すると、 詳しい手順が見つかります。

❌ エラー: "WSL 2 installation is incomplete" (Windows)

WSL 2 installation is incomplete.
The WSL 2 Linux kernel is now installed using a separate MSI update package.

🔍 原因

WSL 2のLinuxカーネルがインストールされていません。

✅ 解決方法

  1. Linuxカーネル更新プログラムをダウンロード
    公式ダウンロードURL:
    https://aka.ms/wsl2kernel
  2. ダウンロードしたMSIファイルを実行

    wsl_update_x64.msiをダブルクリックしてインストール

  3. PCを再起動
  4. Docker Desktopを再度起動

❌ Docker Desktopが起動するけどコマンドが動かない

✅ 解決方法

  1. Docker Desktopの再起動
    • Docker Desktop の設定からQuit（終了）
    • もう一度起動
  2. 完全なリセット
    • Docker Desktop設定 → Troubleshoot → Reset to factory defaults
    • ※ 全てのコンテナやイメージが削除されるので注意
  3. 再インストール
    • Docker Desktopをアンインストール
    • PCを再起動
    • もう一度インストール

💡 それでも解決しない場合

  • Docker公式ドキュメントのトラブルシューティング: https://docs.docker.com/desktop/troubleshoot/overview/
  • エラーメッセージをそのままGoogle検索すると、同じ問題の解決法が見つかることが多いです
  • Docker公式フォーラム: https://forums.docker.com/
