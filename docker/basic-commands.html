📦 イメージ管理コマンド

docker pull - イメージの取得

Docker Hubなどのレジストリからイメージをダウンロードします。

docker pull nginx
💡 最新版（latest）を取得
docker pull nginx:1.24
💡 特定バージョンを指定して取得
🔍 仕組みの理解

イメージはレイヤー構造で構成されており、すでに持っているレイヤーは再ダウンロードされません。これにより効率的にイメージを取得できます。

docker images - イメージ一覧の表示

ローカルに保存されているイメージの一覧を表示します。

docker images
REPOSITORY TAG IMAGE ID CREATED SIZE
nginx latest a6bd71f48f68 2 weeks ago 187MB
ubuntu 22.04 ca2b0f26964c 3 weeks ago 77.9MB
REPOSITORY

イメージの名前

TAG

バージョンやバリエーション

IMAGE ID

イメージの一意な識別子

SIZE

イメージのサイズ

docker rmi - イメージの削除

不要なイメージを削除してディスク容量を節約します。

docker rmi nginx
💡 イメージ名で削除
docker rmi a6bd71f48f68
💡 IMAGE IDで削除
⚠️ 注意

コンテナが使用中のイメージは削除できません。先にコンテナを削除する必要があります。

docker search - イメージの検索

Docker Hub上のイメージを検索します。

docker search mysql

🚀 コンテナ操作コマンド

docker run - コンテナの作成と起動

最も重要なコマンド。イメージからコンテナを作成して起動します。

基本形
docker run nginx
よく使うオプション
docker run -d nginx

-d（detach）: バックグラウンドで実行

docker run -p 8080:80 nginx

-p（port）: ポート公開（ホスト:コンテナ）

docker run --name my-nginx nginx

--name: コンテナに名前を付ける

docker run --rm nginx

--rm: 停止時に自動削除

docker run -it ubuntu bash

-it: 対話モード（-i: 標準入力, -t: 疑似TTY）

🔍 組み合わせ例
docker run -d -p 8080:80 --name web nginx

「webという名前で、nginxをバックグラウンド実行し、ホストの8080番をコンテナの80番に接続」

docker ps - コンテナの確認

実行中のコンテナを確認します。

docker ps
💡 実行中のコンテナのみ表示
docker ps -a
💡 停止中も含めすべて表示

docker start / stop - コンテナの開始・停止

既存のコンテナを起動・停止します。

docker start my-nginx
💡 停止中のコンテナを起動
docker stop my-nginx
💡 実行中のコンテナを停止
💡 runとstartの違い
  • docker run: 新しくコンテナを作成して起動
  • docker start: 既存のコンテナを再起動

docker rm - コンテナの削除

不要なコンテナを削除します。

docker rm my-nginx
💡 停止中のコンテナを削除
docker rm -f my-nginx
💡 実行中でも強制削除

docker exec - 実行中コンテナでコマンド実行

起動中のコンテナ内でコマンドを実行します。

docker exec -it my-nginx bash
💡 コンテナ内でbashを起動（対話モード）
docker exec my-nginx ls /usr/share/nginx/html
💡 コンテナ内でコマンドを実行

🔧 システム管理コマンド

docker logs - ログの確認

コンテナの標準出力・エラー出力を確認します。

docker logs my-nginx
💡 すべてのログを表示
docker logs -f my-nginx
💡 リアルタイムで監視（-f: follow）
docker logs --tail 50 my-nginx
💡 最新50行のみ表示

docker inspect - 詳細情報の確認

コンテナやイメージの詳細な設定情報を表示します。

docker inspect my-nginx
💡 JSON形式で詳細情報を表示

docker system - システム情報とクリーンアップ

Dockerシステム全体の情報確認や不要なデータの削除を行います。

docker system df
💡 ディスク使用量を確認
docker system prune
💡 未使用のデータを削除
⚠️ 注意

docker system pruneは停止中のコンテナ、未使用のネットワーク、ダングリングイメージを削除します。必要なものまで消さないよう確認してから実行してください。

docker version / info - バージョンとシステム情報

docker version
💡 Dockerのバージョン情報
docker info
💡 Dockerシステムの詳細情報

✍️ 実践演習

これまで学んだコマンドを使って、実際に操作してみましょう。

課題1: Nginxコンテナの起動と確認

1. nginxイメージをpullする
2. 「my-web」という名前で、ポート8080でnginxを起動する（バックグラウンド）
3. ブラウザで http://localhost:8080 にアクセスして動作確認
4. docker psで実行中のコンテナを確認
5. docker logsでログを確認

課題2: コンテナ内でのコマンド実行

1. docker execで「my-web」コンテナ内に入る
2. コンテナ内で「ls /usr/share/nginx/html」を実行してファイルを確認
3. 「exit」でコンテナから抜ける

課題3: クリーンアップ

1. 「my-web」コンテナを停止する
2. docker ps -aで停止したコンテナも含めて確認
3. 停止したコンテナを削除する
4. nginxイメージを削除する
✅ 習得目標

これらの演習を通じて、Dockerの基本的なライフサイクル（pull → run → exec → stop → rm）を体で覚えましょう。
